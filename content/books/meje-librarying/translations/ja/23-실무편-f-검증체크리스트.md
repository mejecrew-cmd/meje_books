# 第6編　実務編 F．検証チェックリスト

このチェックリストは、Vault を次の作業へ引き渡す前に確認する最低基準である。各項目を `PASS / WARN / FAIL` で判定する。`FAIL` が一つでも残る場合、運用向け handoff は行わない。

## 判定基準

| 判定 | 意味 | 対応 |
|---|---|---|
| PASS | 基準を満たす | 次の段階へ進む |
| WARN | 使用可能だが記録と後続対応が必要 | handoff メモに残す |
| FAIL | 消費作業へ渡すとエラーの危険が大きい | 修正後に再検証する |

## 第1段階：自動チェック

| 項目 | 基準 | 判定 | メモ |
|---|---|---|---|
| 第1段階の入力カバレッジ | 主要入力文書の前・中・後半の出典がすべてある |  |  |
| 第1段階の9分類網羅 | 全9分類が各1件以上。例外は profile に理由を記録 |  |  |
| 第2段階の代表キーワード重複 | 0件 |  |  |
| 第2段階の必須英語名欠落 | 翻訳対象プロジェクトでは0件 |  |  |
| 第2段階の版状態「未定」率 | 5%以下 |  |  |
| 第2段階の世界観軸「未定」率 | 10%以下 |  |  |
| enum 違反 | category、worldbuilding_axis、version の許容値違反が0件 |  |  |
| 第3段階のファイル数 | 第2段階 CSV の見出し語行数と `.md` 見出し語ファイル数が一致 |  |  |
| YAML 解析 | frontmatter の解析失敗が0件 |  |  |
| ファイル名衝突 | 代表キーワードのファイル名衝突が0件 |  |  |
| 切れたリンク | wikilink の対象欠落が0件、または stub がある |  |  |
| self-link | 自分自身を指すリンクが0件 |  |  |
| LLM 完成率 | 運用向け handoff 基準は95%以上 |  |  |
| manifest 整合 | manifest の進捗率と実ファイル状態が一致 |  |  |

## 第2段階：手動サンプル

サンプルは最低30件を推奨する。hub、中間階層、leaf、9分類、各世界観軸が混ざるように選ぶ。

| 項目 | 基準 | 判定 | メモ |
|---|---|---|---|
| 同義語グループ | 代表キーワードと別名が同一対象を指す |  |  |
| 分類の適合性 | 9分類が物語上の機能に適合する |  |  |
| 世界観軸の適合性 | 世界観軸が IP 構造に適合する |  |  |
| 定義の品質 | 一、二文で自己完結した意味を示す |  |  |
| 詳細説明の品質 | 関連見出し語と機能文脈が分かる |  |  |
| LLM 記述の品質 | hub のトーン基準と衝突しない |  |  |
| 翻訳ノート | 翻訳リスクのある見出し語に十分な指針がある |  |  |
| 出典の逆追跡 | 原文または第1段階の出典へ戻れる |  |  |
| 過剰リンク | 関連キーワードが無意味に膨らんでいない |  |  |
| センシティブな衝突 | 刊行 LOREBOOK または公式本文と衝突しない |  |  |

## 第3段階：統合ビルドチェック

| 項目 | 基準 | 判定 | メモ |
|---|---|---|---|
| Obsidian 閲覧 | Vault が正常に開く |  |  |
| グラフ表示 | 主要 hub とリンク網が異常に孤立していない |  |  |
| 消費 skill の読み取り | 執筆・ロアブック・翻訳作業で見出し語を参照できる |  |  |
| 補助文書 | manifest 登録済みの必須 `aux_docs` が存在する |  |  |
| インデックス | 全見出し語一覧または index が最新 |  |  |
| handoff 記録 | 次の担当者が状態を理解できる |  |  |

## Handoff チェックボックス

```markdown
## Handoff Checklist

- [ ] 自動チェックの FAIL が0件
- [ ] 手動サンプルの FAIL が0件
- [ ] WARN 項目の理由と後続対応を記録
- [ ] LLM 完成率95%以上
- [ ] 必須補助文書が存在
- [ ] manifest の `ready_for_consumption` を確認
- [ ] 刊行 LOREBOOK または公式本文と衝突なし
- [ ] 次の消費作業とバージョン番号を共有

handoff_status: ready_for_consumption / ready_with_warnings / not_ready
handoff_date:
handoff_owner:
handoff_note:
```

## 運用判断

個人練習用 Vault は、WARN や一部未完成を抱えたままでも学習目的で使える。ただし、消費 skill に渡す運用用 Vault はこのチェックリストを通過しなければならない。基準を緩めれば、前工程で節約した時間を後工程のエラーで何倍にもして支払うことになる。
