# 第6編　実務編A　プロジェクトプロファイル・テンプレート

プロジェクトプロファイルはLibraryingの最初の固定点です。第1次抽出前にこのファイルを作れば、以後の第2次統合の世界観軸、第4次叙述のトーン、検証基準がぶれません。

下の書式をそのままコピーして `{プロジェクト名}_profile.md` として使います。

```markdown
# {プロジェクト名} - Libraryingプロジェクトプロファイル

作成日: {YYYY-MM-DD}
作成者: {名前またはチーム}
バージョン: v0.1
状態: draft / active / locked

- プロジェクト名:
- IP種別: 小説 / ウェブ小説 / ウェブトゥーン / ゲーム / ルールブック / 映像 / 混合
- Libraryingの目的: 新規構築 / 既存Vault更新 / 翻訳準備 / ロアブック制作 / B2B共有
- 入力ディレクトリ:
- 入力文書数:
- 入力総量の推定:
```

## 使用メモ

プロファイルは一度書いて捨てる文書ではありません。世界観軸を変える、入力文書群を追加する、第4次トーン基準を変える時には、このファイルの決定ログへ残します。そうして初めて第3次manifestと第10章の検証で「なぜこの状態になったか」を追跡できます。
