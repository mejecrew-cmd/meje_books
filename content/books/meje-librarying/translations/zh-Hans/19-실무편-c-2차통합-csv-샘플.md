# 第6篇　实务篇 C．第二阶段整合 CSV 示例

本实务篇展示第一阶段分散的行如何在第二阶段整理为词条单元。第二阶段的目标不是写漂亮句子，而是稳定确定 Vault 所依赖的代表关键词、别名、世界观轴和相关关键词。

示例 IP 为**“努比之地”**。以下接续展示实务篇 B 的第一阶段抽取示例如何在第二阶段归并为词条。

## 十三列定义

| 列 | 说明 |
|---|---|
| IDX | 词条行号 |
| 分类 | 九种分类之一 |
| 代表关键词 | 成为 Vault 文件名的官方词条 |
| 世界观轴 | Profile 允许的轴 |
| 异称_别名 | 指向同一对象的替代表记 |
| 英文名 | 用于翻译或国际化的表记 |
| 罗马字 | 韩文词条的罗马字表记 |
| 定义 | 一至两句的自足定义 |
| 详细说明 | 包含关系与运作脉络的短说明 |
| 相关关键词 | 按代表关键词列出的相关词条 |
| 版本状态 | 现行 / 过去 / 废弃 / 双重表记 / 未定 |
| 翻译注释 | 翻译判断与注意事项 |
| 出处列表 | 第一阶段行的出处集合 |

## 第二阶段审核要点

| 项目 | 合格标准 |
|---|---|
| 代表关键词重复 | 相同代表关键词不应存在于两行。 |
| 世界观轴 | 仅使用 Profile 允许值。 |
| 别名 | 同一别名若指向不同词条，进行人工审核。 |
| 相关关键词 | 按代表关键词书写，不随意加入别名链接。 |
| 版本状态 | 未定比例保持在全体 5% 以下。 |
| 翻译注释 | 有禁译语、原语并列或首次出现解释需求时明记。 |

## 第二阶段由人最终决定的事

- 同义词组
- 代表关键词
- 新设或变更世界观轴
- 版本状态
- 翻译判断

自动化可以创建候选，但这五项会改变 IP 的意义，因此必须由人批准。
