# 第9章　第4阶段叙述写作：为 Vault 注入生命

在东方王朝世界观[「雨师府」](https://whtdrgongmail.github.io/MEJEwiki/worlds/h9-usabu/)中查找“祈雨祭”词条，会看到“在干旱时祈求降雨的国家仪礼”这个定义，详细说明则整理了仪礼的顺序、条件和结果。但作家只凭这些能写出那一幕吗？若要写背对龟裂稻田、第一次向干枯天空祈雨的正午景象，乃至不知是茫然还是迫切的感觉，仅有定义和详细说明还不足以下笔。

第4阶段叙述写作，正是写出这些叙述的阶段。

## 为什么需要叙述：定义不足的理由

回想第5章做出的定义：“祈雨祭是在干旱时祈求降雨的国家仪礼，成为考验王之德行和正统性的舞台。”它准确、自足，在其他词条详细说明中被引用时可立即理解，是好的定义。

但这还不够。想在小说或电视剧中描绘初次举行祈雨祭的人物的作家，读完这个定义仍不能立刻写出场面。仪礼有什么氛围、带来什么感官经验、此 IP 世界中的人们如何接受它、最初奉行仪礼的人此后如何改变，全部都留在定义之外。

此时需要的是**场面化切入点**。场面化切入点不是把词条直接转为小说句子或分镜的完成稿，而是定义和详细说明抓不住的创作落点。它比事件小、比镜头或画格大：不是整个“祈雨祭”事件，而是背对龟裂稻田初次望向干空的正午；王在祈雨、群众却已在沉默中预感失败的场面；祭官祝文结束后连一丝风也不起的瞬间。

这个单位在各媒体有不同名称。网络小说中是场面或视点场面；Webtoon 中是由多个画格组成的场面；电影和电视剧中是 scene 或 sequence；游戏中会变成事件场景、过场动画、任务场面，有时是关卡的核心瞬间。名称不同，功能却相同：把词条概念落到读者、观众或玩家实际能经验的瞬间。

场面化切入点有若干类型。第一是**感官型**：龟裂田地的泥土气味、正午白光、覆尘祭坛、不发一声的群众，捕捉词条的感官温度。第二是**行为型**：祭官举起没有水的器皿、王跪在干地上、百姓不看天空而看水门，这些是词条以身体行为显现的瞬间。第三是**关系型**：谁在谁面前执行仪礼、谁相信或怀疑它、视线的位置如何分开。

第四是**冲突型**。当祈雨祭不只是祈雨仪礼，而是考验王之德行和正统性的舞台时，词条不再是说明，而成为压力。第五是**转换型**：仪礼结束后因什么也没发生而不再相信天空的人，或因一丝迟来的雨而重新巩固权力秩序的瞬间。不是每次都须填满这五种，但重要 hub 词条至少应有一个场面化切入点，作家才能把它转为实际场面。

百科全书编纂学长期区分定义和解说。定义以逻辑说明类别和特性，解说则讲述对象实际如何运作、置于什么脉络、何处有趣。好的百科条目从定义开始，接着解说。Librarying 中的 `%% LLM %%`区块就是盛放解说的地方。若定义和详细说明是逻辑性技术，LLM 区块就是显示词条在 IP 世界中活着存在的叙述。

有了这种叙述，两件事会改变。第一是 IP 创作者参考 Vault 时的经验：只有定义的 Vault 是词典，填满 LLM 区块的 Vault 是 IP 世界的记忆库；翻词典和漫步记忆库是完全不同的经验。第二是 AI 工作伙伴把 Vault 当作上下文使用时的质量。AI 在 Vault 参照“祈雨祭”时，只有定义便会给出一般知识层次的回答；有 LLM 区块叙述则会以此 IP 独有质地回答。Vault 的密度决定产出的密度。

我在游戏中最鲜明地看见，一个个词条的叙述汇集本身就能让世界活起来。FromSoftware 的《艾尔登法环》和《黑暗之魂》节省主线过场动画，转而在武器、防具、护符等道具的每一行说明中放入世界史片段，玩家收集碎片并在脑中重建世界。我把这作为第4阶段叙述的模型：若每个词条都如同这样的说明文搭建，即使作家不按顺序零散阅读 Vault，碎片也会聚成活着的世界。《空洞骑士》也遵循相同原理，随着击败敌人而填满的世界内图鉴“猎人日志”在 168 个项目中记下敌人信息、猎人的视线和世界历史，使图鉴本身成为解开世界的入口。

## 第4阶段写作的结构

第4阶段写作分四步进行。

首先**亲自写 hub 词条。** 制作人从 100 到 150 个 hub 中亲自写五到十个代表词条，这些叙述成为此后全部写作的语调标准。词条沉重的 IP 只写五个也可能足够；语调不易掌握的 IP 则需要写近十个。

接着**把其余交给 AI 工作伙伴。** 将剩余 hub、中间词条和 leaf 分为批次委派；批次大小和同时处理数不是固定值。词条较轻可在一批放更多，混入主角或核心装置等比重大的词条便减少批次大小。

**经过五阶段检阅。** 不把委派结果原样注入文件，只有通过五阶段检阅的叙述才进入 LLM 区块。

最后**更新 manifest。** manifest 是摘要构建结果的文件；每一批完成时记下进度，工作中断后重启便能确认该从哪批继续。

## 亲自写五到十个 hub：寻找声音的过程

为什么要亲自写 hub？因为写作的声音不能靠说明交接，只能在实际写作中被确立。制作人先挑选最能代表世界观气质的五到十个词条，逐一完成它们。

选择时不以人气或资料量为唯一标准。应当涵盖作品的不同密度：一个人物、一个空间、一个组织或装置、一个事件和一个世界规则。它们共同构成判断后续叙述是否“像这个世界”的基准。

每写完一个词条，都要反问几个问题：第一段是否给读者留下了可感知的第一印象？事实是否被放进了人物或空间的行动之中？术语解释是否打断了阅读？与相关词条的链接是否自然？这些答案会逐步把抽象的语气变成可复用的准则。

## 人物词条的四层篇幅

人物词条不是把履历按年份排开，而是让读者先遇见一个正在存在的人。四层结构可以让分量随角色的重要程度自然伸缩。

第一层是开场场景：用一个动作、习惯或关系把人物带到眼前。第二层是外在印象与位置：他在这个世界中被如何看见，又承担什么功能。第三层是来历和转折：只选择解释当前人物状态所需的过去。第四层是关系与后续：指出他与其他人物、事件、地点之间仍在变化的连线。

主角可以完整使用四层；主要人物可缩短第三层；功能性人物则保留开场、位置和必要的关系。篇幅不是奖赏，而是信息承担量的结果。

**主角**

主角词条需要使读者即使没有读过原作，也能理解其欲望、冲突和行动方向。关键不是把所有设定写全，而是选取能让人物主动起来的事实。

**主要人物**

主要人物以其与主角或核心事件的关联为重心。把独立履历压缩到必要程度，留出关系发生变化的位置。

## 人物以外词条的结构化

地点、组织、道具、制度和概念也需要叙述，但它们的生命感来自使用方式，而不是人格化修辞。地点可从人进入其中时感到的秩序写起；组织可从其成员必须遵守的规则写起；道具可从它改变了谁的选择写起。

每个非人物词条都应回答三件事：它是什么；它在世界中如何发挥作用；读者下一步应连接到哪里。定义放在第二或第三句通常比放在标题下更自然。

## 故事摘要：第4阶段的核心输入

故事摘要不是宣传文案，也不是逐场景复述。它是向写作中的 AI 工作伙伴提供因果与情绪脉络的核心输入：谁在什么条件下做了选择，选择造成什么变化，变化又如何把故事推向下一处。

摘要应保留转折、关系变化、世界规则被触发的时刻，以及角色行动的代价。若只列出设定名词，AI 会写出正确却没有方向感的说明；若只写情节，则会失去词条之间的参照框架。两者必须同时存在。

## 委派结构：批次与并行处理

委派不是把整座 Vault 一次交出去。先按词条类型和重要度划分批次：hub、人物中间词条、地点与组织、概念与道具、leaf。每批都附上适用的故事摘要、相邻词条、长度和语气要求。

并行处理也必须保留边界。共享同一核心人物或同一关键事件的词条不应同时独立生成，否则关系表述容易互相冲突。先完成基准 hub，再让后续批次以它们为参照，能够显著降低修订成本。

## 委派后的五阶段检阅

**五阶段检阅**把“生成完成”与“可以收录”分开。

第一阶段检查事实：名字、关系、时间、地点、装备与事件顺序是否正确。第二阶段检查声音：它是否遵循已经确立的叙述温度，而非回到百科式清单。第三阶段检查结构：开场、定义、发展和链接是否各自承担作用。第四阶段检查连接：双向链接、别名、相关词条和摘要输入是否一致。第五阶段检查注入：确认 frontmatter、LLM 区块边界和 manifest 更新均无遗漏。

任何阶段出现问题都回到相应步骤修订；不以“整体读起来还可以”为理由跳过事实和链接的核对。

## 沿着一个检阅周期走一遍：leaf 词条 [[堰的水门]]

以 leaf 词条“堰的水门”为例。初稿可能准确说明它控制水流，却没有说明它为何会在故事中被看见。事实检阅时，确认它所属的位置、被谁维护、在何种事件中开启或关闭。

声音检阅时，将开头从定义换成具体处境：值守者在水压升高时听见门轴的声响，随后再说明水门的功能。结构检阅时，删去与上位地点词条重复的地理细节，把必要的技术事实保留为一两句。

连接检阅时，补上所属水道、管理组织、相关事故与维护人物的链接。最后注入时，确认它只扩展应扩展的 LLM 区块，且 manifest 的该批进度已更新。一个小词条也以同样的过程获得可追溯性。

## 第4阶段的时间预算

第4阶段的时间不应只按词条数量估算。亲自写 hub、准备摘要、首轮委派、五阶段检阅和返工都需要预算；越是核心词条，检阅所占比例越高。

可以先用少量 hub 试运行，记录一篇从写作到注入所需的实际时间，再据此安排后续批次。若试运行中反复出现同类错误，应先修订提示词、摘要或词条模板，而不是机械加快生成速度。

## 把一个 hub 跟到最后：语气标准如何形成

选择一个 hub，从开场场景写到链接完成，最容易看见语气标准是怎样出现的。第一稿往往会过度解释；第二稿开始知道哪些术语可以后置；到第三稿，人物或空间的动作会自然承担信息。

随后把这个 hub 与相邻的两个 leaf 词条并读。若它们只是在重复定义，说明 hub 没有为后续写作提供方向；若读者能沿着链接感到世界正在扩展，语气才真正成为标准。

## 第4阶段完成时：Vault 仍在呼吸

第4阶段完成，不意味着每个词条都写得同样长，而是 Vault 已经拥有一致的叙述重力。读者从一个词条跳到另一个词条时，既能获得需要的事实，也能感到这些事实属于同一个正在运转的世界。

此时的 Vault 不再只是被整理过的资料仓库。它能让制作人继续补写、让 AI 工作伙伴继续协作，并让未来的读者找到进入世界的路径。

## 写完第4阶段之后

第4阶段把前面建立的分类、关系和摘要转化为可阅读的文本。接下来的工作不是放松检阅，而是让这些文本在持续更新中保持可信：新增资料先回到来源和结构，再进入叙述。

亲自写五到十个 hub，先把声音立起来；再以批次委派和五阶段检阅，让整座 Vault 保持同一种生命感。
