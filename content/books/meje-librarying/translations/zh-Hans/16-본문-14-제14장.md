# 第14章　面对空的 Vault：本书想传达的事

回到第1章的第一句话。无法用一个词解释《Star Wars》。原力、绝地、光剑、达斯·维达、反抗军和银河帝国不是孤立的点，而是彼此相连的关系网络；那个整体才是《Star Wars》。一个 IP 就是构成它的所有词汇形成的关系网络。

本书用十四章引导有意识地建立这张关系网的工作。从四阶段管线、九种分类和世界观轴，到幂等规则、验证和运营循环，都是把散落在 IP 脑海中的关系网，取出为可掌握的 Obsidian Vault 资料的程序。

现在读者桌上摆着十三列 CSV 格式、九分类指南和仍然空白的 Markdown 文件。文件之间还没有连线，但读者已经知道：填下第一行的瞬间，第一阶段抽取便开始；这一行增长为数百行，在第二阶段整合为词条，第三阶段变为 Markdown 文件，第四阶段补上叙述。点与点之间将织起连线，一个 IP 的关系网地图会在屏幕上展开。

## 既有创作惯例与 Librarying 的差异

过去并非没有运营 IP 的办法。作家个人的设定笔记、团队共享文档、Google 表格和 wiki 网站都以各自的方式管理 IP 信息。

但这些方式共有四个限制：格式不结构化、依赖单一作者、缺乏版本管理、难以移交。设定笔记由作者任意书写，人能读懂，机器却难以读取和处理；搜索、一致性检查和在其他工具中复用都很困难。真实设定往往在作者脑中，文档只保存一部分，因此作者不在时许多信息会变得不确定。IP 发展时设定虽然改变，却难以追踪哪个版本最新；第三版改过的内容，第二版设定仍可能留在别处。移交给其他团队或多年后重新处理时，大量设定只存在原作者记忆中，因而难以传承。

Librarying 是对这些限制的方法论回应。十三列 CSV 与 Markdown 文件的结构预先确定，机器可以读取，自动化工具可以处理，其他工具也可复用。Vault 不在一人脑中，而在文件系统中；第一次接触 IP 的人打开 Vault 也能确认官方设定。frontmatter 的版本状态列和出处列表列保存每个词条的版本历史，幂等规则则在更新时保护已有工作。交付一个 Vault 文件夹，就会同时交付 IP 的官方词汇与世界观结构。

这正是 Librarying 改进既有惯例的地方。它不是成为更好作家的方法，而是建立能更好管理 IP 的系统的方法。

## 本书想传达的事

若将十四章压缩成一句话：Librarying 不是用 Obsidian 制作 wiki，而是制作人建立一份能运营整个 IP 的唯一原本。

差异不在容器，而在切分方式。即使使用相同的 Obsidian，因为以九分类和世界观轴两套坐标进行切分，这个 Vault 便不是用于搜索的字典，而是用于运营的原本。新短篇、新角色、外语翻译和百科全书，都会通过这一册向外产生。

因此 Vault 不是完成时便结束的资料。各项工作产生的新词汇会通过回流返回，幂等规则保护已经写好的叙述，并在其上叠加下一轮。只要回流没有停止，IP 就仍然活着。

## 制作人的角色：做决定的手

本书始终强调一件事：制作人不是作家。

作家是写作的人；制作人则是一边写、一边管理、委派、审核和重写的人。写作只是他众多工作中的一项。Librarying 是制作人的工作：设计四阶段管线，决定要把什么、怎样委派给 AI 工作伙伴，审核结果并运营 Vault。它包括直接写作阶段（直接写 hub）、委派阶段（leaf 叙述）和确认阶段（验证）。

第2章说过：“做决定的手是人，纺出内容的手是 AI。”读完十四章，读者应能具体感到这句话怎样发挥作用。九分类应放入哪一类由人决定，世界观轴由人设计，直接写 hub、马上修正 P0 问题、通过回流让 Vault 持续成长，也都是人的职责。AI 产出但由人审核通过的内容填满 Vault；AI 出错但由人修正的部分形成 Vault 的质量。当这种分工运作良好时，一个人难以独自承担的规模也能在现实时间内完成。

## 写给开始第一周期的读者

请从小型 IP 开始。第一周期不必以 1,500 个词条为目标；只选 5 到 10 份核心文档，先做出 300 到 500 个词条的 Vault。目标是完整经历一次四个阶段；与其从一开始就试图完美处理大型 IP 而中途停止，不如把小型 IP 做到最后，价值大得多。

初次练习时，世界观轴可以轻量地从三到四个开始。但进入运营用 Vault 就不同。运营工作流应在 profiling 阶段确定五到七个核心轴，再加上“常规”和“未定”；第二阶段整合与验证都在这些允许值范围内运行。这不是要从一开始就猜中完美的轴名，而是至少应在工作前写下将要测试哪些轴。

练习用的第一个 Vault，第二阶段整合不必完美。若文件目标是熟悉完整流程，完成度 85 到 90% 也可以推进到第三、第四阶段；但要交给其他工作的运营用 Vault，合格线不同。同义词组、代表关键词、世界观轴和版本状态必须达到约 95%，并通过自动验证和 handoff 检查。等待完美而始终无法完成 Vault 是问题；把练习用的宽松误认为运营标准也是问题。

第四阶段请先直接写几个 hub。小型练习 Vault 写三到五个即可建立调性标准，运营用首轮则建议五到十个。那些叙述会成为后续写作的基准。许多人试图独自写完全部 hub，结果使第四阶段停滞；重点不在数量，而在先建立足够清晰、可交给 AI 工作伙伴的文体与观点范例。

完成第一个 Vault 后，请在第二天再次打开。经过一夜，同一个 Vault 会以不同的眼光显现：遗漏之处和错误连接会浮现。它们会成为第一份 P2 问题清单，原样带入下一周期。

## 延伸到姐妹文章的内容

如何运用 Librarying 建立的 Vault 写短篇，将在《Storytelling 100》中讨论。Vault 的人物词条会成为角色表，世界观轴会成为日常片段素材的轴。

如何在翻译工作中使用 Vault，将在《Seoyeongak》中讨论。翻译注释列、异称列和别名列会成为翻译一致性的资料。

如何从零塑造角色时参照 Vault 的世界观轴与人物关系网，将在《New Character Build》中讨论；这正是第1章介绍为本书姐妹文章的那篇。如何把 Vault 的 1,500 个词条汇集为一本出版百科全书，将在《LOREBOOK》中讨论。

这些文章与本书在同一条管线中彼此连接。

## 面对空的 CSV

在空白 CSV 文件中输入第一行的瞬间，就是 Librarying 的开始。

`IDX`、`分类`、`关键词`、`说明`、`出处`。第一行填入标题，下一行放入第一个关键词，第一阶段抽取由此开始。

那个关键词经过数百行，在第二阶段整合中被归入代表关键词，第三阶段变成 Markdown 文件，第四阶段写入叙述。文件在 Obsidian Graph View 中成为一个节点，链接连接到节点。节点累积成关系网，而这张关系网就是 IP 的世界观地图。

起初空白的地图，如今已填满 1,500 个点和数千条连线。人物在其中生活，场所存在，事件发生，装置运行，概念流动；这个 IP 的世界已展开成一张地图。

正如无法用一个词解释《Star Wars》，读者的 IP 也无法用一个词解释。这张关系网现在只存在于读者脑中。

阅读到此结束。回到桌前，写下空白 CSV 的第一行。IP 走到屏幕上，只能从那一行开始。
