# 第6章　两个分类基准——九分类与世界观轴

若一个 IP 只有一个分类基准，会发生什么事？打开[“共同”](https://whtdrgongmail.github.io/MEJEwiki/worlds/h1-gongdong/)的人物分类，协会干部、攻略地下城的觉醒者、一般市民聚在一起。三者都是人物。然而，光靠人物分类，无法知道三者属于协会领域、地下城领域还是日常领域。同样限制出现在任何 IP。

本书常用的示范 IP 是[“菌丝海”](https://whtdrgongmail.github.io/MEJEwiki/worlds/h4-gyunsahae/)。覆盖行星的菌丝网是魔法、记忆、沟通的媒质，种族依与何种菌共生而分开，是一个 SF 生态世界。以这个 IP 一路追到最后，便能清楚看见何谓正交，以及两个分类基准如何咬合。

Librarying 使用九分类与世界观轴两个分类基准。两者彼此正交。九分类问“此关键词在叙事中有何功能”，世界观轴问“此关键词属于我们世界观的哪个结构”。每个标题词对两个问题各有一个答案，这两个答案就是坐标。

## 若分类基准只有一个

以武功系于星座的江湖世界“星罗江湖”来看只有九分类时会发生什么事。打开人物分类，正派掌门人、邪派杀手、读天文的观相家、游走江湖的浪人都在同一类。四人都在故事里作为行动者运作，确实都是人物。

但光靠人物分类，很难知道四人彼此有何关系，或在这个 IP 世界观中属于哪个领域。正派掌门人跨在正邪派领域与率领一个门派的门派领域；观相家接触武功系于星的天文领域；浪人则属于不隶属任何门派的江湖领域。同样是“人物”，在 IP 中却分布于完全不同的世界观领域。

若此区别未在 Vault 里表达，有人搜索“显示所有天文领域的人物”便得不到答案，只能打开整个人物分类逐一用眼挑选。标题词若有 1,500 个，这极其麻烦；世界观轴作为第二分类基准加入，正好解决这个问题。

## 分面分类：两个独立基准的历史

同时使用两个独立分类基准不是 Librarying 的发明。信息科学与图书馆学有一种称为**分面分类**（Faceted Classification）的方法论。

动物→脊椎动物→哺乳类→食肉类→猫科，这样按一个基准往下分层的是传统层级分类。它直观，但只能使用一个基准，无法完成“住在非洲的哺乳类”这样同时套用地域与分类群两个基准的搜索。

分面分类不同之处在于，将一个对象分成多个独立分面。以一本书为例，主题是历史、地域是韩国、时代是朝鲜、形式是小说；各面独立附着，既可按任一分面搜索，也可如“朝鲜时代韩国历史小说”般同时套用四个分面。

Obsidian 的 tags 系统实现此分面分类，Librarying 的九分类与世界观轴便是把此系统用作两个独立分面。

## 正交是什么

在数学中，两个轴正交是指两者独立运行，一轴上的值不决定另一轴的值。若 X 轴决定“朝哪个方向”，Y 轴决定“有多高”，两轴即为正交：朝东也可以低或高；高也可以朝东或朝西。

以“菌丝海”看九分类与世界观轴的正交。假设此 IP 的世界观轴定为“菌丝网”“种族”“共生”“身份”“地形”“一般”六轴。“光菌族”在九分类是人物、在世界观轴是种族；“空孢子囊”是物件且属种族；“共生”是概念且属共生；“菌丝接续”与“共享记忆”是装置且属菌丝网；“掌握大节点的支配者”是人物且属身份；“不毛地”是场所且属地形；“发光菌丝”是场面调度且属地形。同一世界观轴“种族”有人物、物件、概念；同一九分类人物有种族轴的光菌族与身份轴的支配者。一个标题词以一个九分类乘上一个世界观轴来坐标化，两坐标不决定彼此即为正交。

## 事件将人物、场所与物件束在一起时

设计世界观轴时常漏掉一个观点：光把人物、场所、物件、事件分开，关系不会充分显现。CIDOC CRM 等整合文化遗产信息的本体论，不把人、场所、物件、时间只看作独立清单，而试图通过事件与行为连接它们，理由正在此处。博物馆文物的意义，来自它曾在哪个时代、哪个场所、经谁的手、置于何种事件中被一并记录。

IP 也一样。一把剑是物件，剑客是人物，比武场是场所；但将剑从谁交给谁、在哪个比武场折断、事件后门派权威如何改变绑在一起的是事件。没有事件，人、场所、物件只是各自清单；事件进入后，三张清单才连成一段世界观记忆。

因此世界观轴不是单纯建立“人物轴”“场所轴”“物件轴”。九分类已把人物、场所、物件分开；世界观轴要显示它们在何种世界观领域与事件流程中相遇。一个 IP 以“王位继承”为轴，另一个以“菌丝网”，又一个以“安置”为轴。好的轴不是将清单切得更细的名称，而是能说明人物、场所、物件、事件同时被束起的场景名称。

## 为何需要世界观轴

只用九分类时，打开人物分类会混合主角、配角以及来自不同世界观领域的人物。叙事功能都同为人物，在 IP 里却属于完全不同的世界观领域。

在 Vault 图谱视图中，人物分类的点聚成一种颜色。要区分“这些人物属于世界观的哪个领域”，需要另一种颜色区分，第二基准便是世界观轴。在 Obsidian 打开 Vault 与图谱视图，可以两种方式为点着色：按九分类，人物蓝色、场所绿色、装置橙色，按叙事功能分开；按世界观轴，各领域有不同颜色，按世界观领域分开。同一个 Vault 如同用两副透镜观看。有了两副透镜，便可搜索“只显示某世界观轴中某九分类的所有关键词”，同时使用两个坐标。

## 世界观轴因 IP 而异

九分类对每个 IP 相同，世界观轴则必须依 IP 设计。现代猎人题材“共同”使用地下城、觉醒、协会、浓度、日常；香气媒介身份与魔法的恋爱奇幻“香宫”使用香、身份、社交界、家门、神圣；武功系于星座的“星罗江湖”使用武功、天文、正邪派、江湖、门派；雨是正统性印记的“雨司府”使用天命、水、官署、权门、民心；死者引导航路的宇宙 SF“丧舆号”使用冥路、航行、家门、安置、次元。各 IP 核心领域全然不同。

同为九分类物件的一物，在 IP 中所属轴也不同。“星罗江湖”的本命星牌属武功；“香宫”的香水瓶属身份；“共同”的等级卡属协会；“丧舆号”的牌位属航行。叙事功能同为物件，依世界观结构却落在不同轴。人物也一样：香宫的大贵族调香师属社交界，雨司府的官署首长属官署，丧舆号的安置院祭司属安置。没有一套正确的世界观轴模式；先确认 IP 置何者于中心再设计轴。

D&D《Forgotten Realms Campaign Setting》的 Magic、Deities、History、Organizations、Life in Faerûn 等章节正是世界观领域而非九分类；每章都包含人物、场所、物件、事件，同一人物常在多章出现。《Eberron Campaign Setting》也是相同方式。Librarying 借用这种领域分类作为世界观轴，并正交叠加九分类，才可搜索“只要信仰领域的人物标题词”。

## 世界观轴的设计原则

**数量是 5 至 7 个。** 太少分类粗糙，太多难以管理，五至七个核心领域是建议范围。

**必须有“一般”与“未定”。** 一般容纳不属特定世界观领域的通用词汇；未定容纳尚不能决定所属轴的标题词。

**归纳决定。** 先扫描标题词，发现自然形成的组合，比先在脑中编轴再把标题词塞入更能形成适合本 IP 的轴。

**不固定，但不随意改。** 每次变更必须记录在配置文件，并重新检查已分配标题词；清单外轴名会在验证中报错。

## 为何必须在配置中先决定

第二次整合世界观轴字段必须有允许值清单。没有清单，AI 会为每个标题词即兴附轴，同一“菌丝网”可能分裂为“菌丝网”“菌丝”“菌丝网世界观”“菌丝界”。配置阶段先固定允许值，委派时一同交付，AI 便只在允许值内附轴，写法得以统一。

配置产物为`{项目}_profile.md`，其中还包含九分类的 IP 特记、辅助参考文件清单、第四次叙事写作的语调指南。它是 Librarying 第一次循环的起点。

## 归纳决定：让标题词说出自己的轴

阅读“菌丝海”文档会看到自然共现的组合：菌丝网、节点、接续、共享记忆；光菌族、腐败菌族、石菌族、孢子游浪族；光菌森林、腐败菌沼泽、石菌山脉、不毛地。这些组合提示菌丝网、种族、地形轴。与何种菌结合决定能力的组合成为共生；大节点聚集权力的组合成为身份；不进入任何组合的通用词汇为一般；再加上未定轴。先发现 IP 的模式，让标题词自然坐进自己的分类，而不是先定名称后强塞。

## 两坐标同时存在时

要收集“菌丝海”菌丝网领域的装置标题词，同时应用世界观轴“菌丝网”与九分类“装置”，便会立刻显示接续和共享记忆。写“丧舆号”航行短篇可将航行×装置相乘；挑选“香宫”社交界场景人物可将社交界×人物相乘。两个正交基准直接产生所需交集。

## frontmatter 的实际样子

第三次骨架构建生成 .md 文件时，其 frontmatter 如下：

```yaml
---
title: 菌丝接续
aliases: [接续, 菌丝网接续]
category: 装置
worldbuilding_axis: 菌丝网
english: Mycelial Linking
romanization: Gyunsa Jeopsok
version: 现行
tags:
  - category/装置
  - axis/菌丝网
  - version/现行
sources:
  - 规则书第3版 p.24
related:
  - "[[共享记忆]]"
  - "[[无菌者]]"
---
```

`category` 是九分类，`worldbuilding_axis` 是世界观轴；它们进入 `tags` 数组，因而可被搜索。第七章会再详细处理这一结构。

## 世界观轴运营的两个陷阱

第一是轴膨胀：每遇到放入一般轴暧昧的标题词就新设轴，六轴会膨胀到十二轴；超过十轴，图谱颜色和搜索筛选都失去可用性。先尝试让既有轴吸收，无法吸收才新设，并记录在配置更新。第二是混淆九分类和世界观轴：菌丝网是世界观领域，装置是叙事功能；同一菌丝网领域可以同时有装置、人物、价值、事件。必须记住两基准正交。

## hub 决定的辅助资料

每个世界观轴中相关关键词被引用最多的标题词是该轴的 hub 候选。查找各轴 hub，再与整体引用前列并置，可显示 Vault 的立体结构中心。比较菌丝网轴与种族轴拥有的 hub 数，可判断 IP 哪个领域发展得更深；若前者四十个而后者五个，便是种族世界观相对未发展的信号，可为下一个创作方向提供资料。

## 手持两坐标系

配置中确定的允许值随第二次整合任务交给 AI 工作伙伴，AI 为各标题词附候选轴，人再验收确定。两个基准的交集搜索呈现一个基准看不见的内容。下一章进入第三次骨架构建：十三列 CSV 自动转换为 1,500 个 Obsidian .md 文件，屏幕上首次展开 1,500 个点与线。
