# Part 6. Practice A. Project Profile Template

A project profile is Librarying’s first fixed point. Creating this file before Stage 1 extraction keeps the worldbuilding axes of Stage 2 consolidation, the tone of Stage 4 narration, and validation criteria from drifting.

Copy the template below as `{project-name}_profile.md`.

## Usage notes

A profile is not a document written once and discarded. When you change worldbuilding axes, add input documents, or change Stage 4 tone standards, record the change in this file’s decision log. Only then can the Stage 3 manifest and Chapter 10 validation trace why the current state came to be.
