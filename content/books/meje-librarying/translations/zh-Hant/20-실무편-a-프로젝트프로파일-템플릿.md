# 第6篇　實務篇 A．專案 Profile 範本

專案 Profile 是 Librarying 的第一個固定點。在第一階段抽取前建立此檔案，後續第二階段整合的世界觀軸、第四階段敘述的調性與驗證基準便不會搖擺。

請直接複製下列格式，作為 `{專案名稱}_profile.md` 使用。

```markdown
# {專案名稱} - Librarying 專案 Profile

建立日期：{YYYY-MM-DD}
建立者：{名稱或團隊}
版本：v0.1
狀態：draft / active / locked

- 專案名稱：
- IP 類型：小說 / 網路小說 / 網漫 / 遊戲 / 規則書 / 影像 / 混合
- Librarying 目的：新建 / 更新既有 Vault / 翻譯準備 / lorebook 製作 / B2B 共用
- 輸入目錄：
- 輸入文件數：
- 輸入總量估計：
- 目標詞條數範圍：1,000～2,000 / 其他
- 目標完成日：
```

## 使用備註

Profile 不是寫一次就丟棄的文件。變更世界觀軸、加入輸入文件群或改變第四階段調性基準時，都要記進此檔案的決定紀錄。如此才能在第三階段 manifest 與第10章驗證中追蹤「為什麼會形成這個狀態」。
