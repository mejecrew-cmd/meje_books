# 第6篇　實務篇 F．驗證檢查清單

本清單是在將 Vault 交給下一項工作前必須確認的最低標準。各項以 `PASS / WARN / FAIL` 判定。只要仍有一項 `FAIL`，就不可進行營運用 handoff。

## 判定標準

| 判定 | 意義 | 措施 |
|---|---|---|
| PASS | 符合標準 | 進入下一階段 |
| WARN | 可使用，但需記錄與後續處置 | 註記於 handoff 備忘 |
| FAIL | 交給消費工作時有高度錯誤風險 | 修正後重新驗證 |

## 第 1 階段：自動檢查

| 項目 | 標準 | 判定 | 備註 |
|---|---|---|---|
| 第1階段輸入涵蓋度 | 主要輸入文件的前、中、後段出處皆存在 |  |  |
| 第1階段九分類涵蓋度 | 九分類皆至少有一筆；例外於 profile 記錄原因 |  |  |
| 第2階段代表關鍵詞重複 | 0筆 |  |  |
| 第2階段必填英文名缺漏 | 翻譯對象專案為0筆 |  |  |
| 第2階段版本狀態未定比例 | 5%以下 |  |  |
| 第2階段世界觀軸未定比例 | 10%以下 |  |  |
| enum 違規 | category、worldbuilding_axis、version 的允許值違規皆為0筆 |  |  |
| 第3階段檔案數 | 第2階段 CSV 詞條列數與 `.md` 詞條檔案數相符 |  |  |
| YAML 解析 | frontmatter 解析失敗0筆 |  |  |
| 檔名衝突 | 代表關鍵詞檔名衝突0筆 |  |  |
| 失效連結 | wikilink 目標缺漏0筆，或存在 stub |  |  |
| self-link | 指向自身的連結0筆 |  |  |
| LLM 完成率 | 營運用 handoff 標準為95%以上 |  |  |
| manifest 一致性 | manifest 進度與實際檔案狀態一致 |  |  |

## 第 2 階段：人工抽樣

建議至少抽樣30項，並混合 hub、中間層級、leaf、九分類與各世界觀軸。

| 項目 | 標準 | 判定 | 備註 |
|---|---|---|---|
| 同義詞組 | 代表關鍵詞與別名指向同一對象 |  |  |
| 分類適切性 | 九分類符合敘事功能 |  |  |
| 世界觀軸適切性 | 世界觀軸符合 IP 結構 |  |  |
| 定義品質 | 以一兩句提供自足的意義 |  |  |
| 詳細說明品質 | 呈現相關詞條與運作脈絡 |  |  |
| LLM 敘述品質 | 不與 hub 語調基準衝突 |  |  |
| 翻譯註記 | 有翻譯風險的詞條具備足夠指引 |  |  |
| 出處逆向追溯 | 可回到原文或第1階段出處 |  |  |
| 過度連結 | 相關關鍵詞未被無意義膨脹 |  |  |
| 敏感衝突 | 不與已發行 LOREBOOK 或官方正文衝突 |  |  |

## 第 3 階段：整合建置檢查

| 項目 | 標準 | 判定 | 備註 |
|---|---|---|---|
| Obsidian 閱覽 | Vault 可正常開啟 |  |  |
| 圖譜顯示 | 主要 hub 與連結網不會異常孤立 |  |  |
| 消費 skill 讀取 | 寫作、Lorebook、翻譯工作可參照詞條 |  |  |
| 輔助文件 | manifest 登錄的必要 `aux_docs` 存在 |  |  |
| 索引 | 全詞條清單或索引為最新 |  |  |
| handoff 記錄 | 下一位工作者能理解狀態 |  |  |

## Handoff 勾選清單

```markdown
## Handoff Checklist

- [ ] 自動檢查 FAIL 0筆
- [ ] 人工抽樣 FAIL 0筆
- [ ] 記錄 WARN 項目的原因與後續措施
- [ ] LLM 完成率95%以上
- [ ] 必要輔助文件存在
- [ ] 確認 manifest `ready_for_consumption` 值
- [ ] 不與已發行 LOREBOOK 或官方正文衝突
- [ ] 與下一項消費工作共享版本號

handoff_status: ready_for_consumption / ready_with_warnings / not_ready
handoff_date:
handoff_owner:
handoff_note:
```

## 營運判斷

個人練習用 Vault 即使帶有 WARN 或部分未完成，也可用於學習；但交給消費 skill 的營運用 Vault 必須通過本清單。放寬標準雖可在前段省時，卻會在後段以數倍錯誤成本償還。
