# 第6篇　實務篇 C．第二階段整合 CSV 範例

本實務篇是展示第一階段分散列如何在第二階段整理為詞條單位的加工範例。第二階段的目標不是寫出漂亮句子，而是穩定確定 Vault 所依賴的代表關鍵字、別名、世界觀軸與相關關鍵字。

範例 IP 為**「努比之地」**。以下接續展示實務篇 B 的第一階段抽取範例如何在第二階段收束為詞條。

## 十三欄定義

| 欄位 | 說明 |
|---|---|
| IDX | 詞條列號 |
| 分類 | 九種分類之一 |
| 代表關鍵字 | 成為 Vault 檔名的官方詞條 |
| 世界觀軸 | Profile 允許的軸 |
| 異稱_別名 | 指向同一對象的替代表記 |
| 英文名 | 用於翻譯或國際化的表記 |
| 羅馬字 | 韓文詞條的羅馬字表記 |
| 定義 | 一至兩句的自足定義 |
| 詳細說明 | 包含關係與運作脈絡的短說明 |
| 相關關鍵字 | 依代表關鍵字列出的相關詞條 |
| 版本狀態 | 現行 / 過去 / 廢棄 / 雙重表記 / 未定 |
| 翻譯註解 | 翻譯判斷與注意事項 |
| 出處清單 | 第一階段列的出處集合 |

## 第二階段檢查要點

| 項目 | 合格基準 |
|---|---|
| 代表關鍵字重複 | 相同代表關鍵字不會存在於兩列。 |
| 世界觀軸 | 只使用 Profile 允許值。 |
| 別名 | 同一別名若指向不同詞條，手動檢查。 |
| 相關關鍵字 | 以代表關鍵字書寫，不任意加入別名連結。 |
| 版本狀態 | 未定比例維持在全體 5% 以下。 |
| 翻譯註解 | 有禁譯語、原語併列或初次出現解說需求時明記。 |

## 第二階段由人最終決定的事

- 同義詞組
- 代表關鍵字
- 新設或變更世界觀軸
- 版本狀態
- 翻譯判斷

自動化可以製作候選，但這五項會改變 IP 的意義，因此由人核准。
