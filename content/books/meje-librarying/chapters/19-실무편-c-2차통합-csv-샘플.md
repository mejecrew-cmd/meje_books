# 제6편. 실무편 C. 2차 통합 CSV 샘플

이 실무편은 1차의 흩어진 행이 2차에서 표제어 단위로 정리되는 모습을 보여 주는 가공 예시다. 2차의 목표는 예쁜 문장을 쓰는 것이 아니라, Vault가 의존할 대표키워드와 별칭, 세계관축, 관련키워드를 안정적으로 확정하는 것이다.

예시 IP는 **「누비의 땅」**이다. 실무편 B의 1차 추출 샘플이 2차에서 어떻게 표제어로 묶이는지를 이어서 보여 준다.

## 13컬럼 정의

| 컬럼 | 설명 |
|---|---|
| IDX | 표제어 행 번호 |
| 분류 | 9분류 중 하나 |
| 대표키워드 | Vault 파일명이 될 공식 표제어 |
| 세계관축 | 프로파일에서 허용한 축 |
| 이칭_별칭 | 같은 대상을 가리키는 대체 표기 |
| 영문명 | 번역 또는 국제화용 표기 |
| 로마자 | 한국어 표제어의 로마자 표기 |
| 정의 | 한두 문장의 자기완결 정의 |
| 상세설명 | 관계와 작동 맥락을 담은 짧은 설명 |
| 관련키워드 | 대표키워드 기준 관련 표제어 목록 |
| 버전상태 | 현행 / 과거 / 폐기 / 이중표기 / 미정 |
| 번역노트 | 번역 판단과 주의사항 |
| 출처목록 | 1차 행들의 출처 묶음 |

## 2차 검수 포인트

| 항목 | 합격 기준 |
|---|---|
| 대표키워드 중복 | 같은 대표키워드가 두 행에 존재하지 않는다. |
| 세계관축 | 프로파일 허용값 안에서만 사용한다. |
| 별칭 | 같은 별칭이 서로 다른 표제어를 가리키면 수동 검토한다. |
| 관련키워드 | 대표키워드 기준으로 적고, 별칭 링크를 임의로 넣지 않는다. |
| 버전상태 | 미정 비율이 전체 5% 이하가 되도록 한다. |
| 번역노트 | 번역 금지어, 원어 병기, 첫 등장 해설이 필요한 경우 명시한다. |

## 2차에서 사람이 최종 결정하는 것

- 동의어 묶음
- 대표키워드
- 세계관축 신설 또는 변경
- 버전상태
- 번역 판단

자동화는 후보를 만들 수 있지만, 이 다섯 가지는 IP의 의미를 바꾸는 결정이므로 사람이 승인한다.
