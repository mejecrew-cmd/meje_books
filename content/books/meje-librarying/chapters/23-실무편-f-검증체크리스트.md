# 제6편. 실무편 F. 검증 체크리스트

이 체크리스트는 Vault를 다음 작업에 넘기기 전에 확인할 최소 기준이다. 항목은 `PASS / WARN / FAIL`로 판정한다. `FAIL`이 하나라도 남아 있으면 운영용 핸드오프를 하지 않는다.

## 판정 기준

| 판정 | 의미 | 조치 |
|---|---|---|
| PASS | 기준 충족 | 다음 단계 진행 |
| WARN | 사용은 가능하나 기록과 후속 조치 필요 | handoff 메모에 남김 |
| FAIL | 소비 작업에 넘기면 오류 위험이 큼 | 수정 후 재검증 |

## 1단계: 자동 단

| 항목 | 기준 | 판정 | 메모 |
|---|---|---|---|
| 1차 입력 커버리지 | 주요 입력 문서의 전·중·후반 출처가 모두 존재 |  |  |
| 1차 9분류 포함도 | 9분류가 모두 1건 이상 존재. 예외는 프로파일에 사유 기록 |  |  |
| 2차 대표키워드 중복 | 0건 |  |  |
| 2차 영문명 필수 누락 | 번역 대상 프로젝트에서 0건 |  |  |
| 2차 버전상태 미정 비율 | 5% 이하 |  |  |
| 2차 세계관축 미정 비율 | 10% 이하 |  |  |
| enum 위반 | category, worldbuilding_axis, version 모두 허용값 위반 0건 |  |  |
| 3차 파일 수 | 2차 CSV 표제어 행 수와 .md 표제어 파일 수 일치 |  |  |
| YAML 파싱 | frontmatter 파싱 실패 0건 |  |  |
| 파일명 충돌 | 대표키워드 파일명 충돌 0건 |  |  |
| 끊어진 링크 | 관련키워드 wikilink 대상 누락 0건 또는 stub 존재 |  |  |
| self-link | 자기 자신을 가리키는 링크 0건 |  |  |
| LLM 완성률 | 운영용 handoff 기준 95% 이상 |  |  |
| manifest 정합 | manifest 진행률과 실제 파일 상태 일치 |  |  |

## 2단계: 수동 샘플 단

샘플은 최소 30개를 권장한다. hub, 중간 층위, leaf, 각 9분류, 각 세계관축이 섞이도록 뽑는다.

| 항목 | 기준 | 판정 | 메모 |
|---|---|---|---|
| 동의어 묶음 | 대표키워드와 별칭이 같은 대상을 가리킴 |  |  |
| 분류 적합성 | 9분류가 서사 기능에 맞음 |  |  |
| 세계관축 적합성 | 세계관축이 IP 구조에 맞음 |  |  |
| 정의 품질 | 한두 문장으로 자기완결적 의미 제공 |  |  |
| 상세설명 품질 | 관련 표제어와 작동 맥락이 드러남 |  |  |
| LLM 서술 품질 | hub 톤 기준과 충돌하지 않음 |  |  |
| 번역노트 | 번역 위험이 있는 표제어에 충분한 지침 존재 |  |  |
| 출처 역추적 | 원문 또는 1차 출처로 돌아갈 수 있음 |  |  |
| 과잉 연결 | 관련키워드가 무의미하게 부풀지 않음 |  |  |
| 민감한 충돌 | 발간 LOREBOOK 또는 공식 본문과 충돌 없음 |  |  |

## 3단계: 통합 빌드 단

| 항목 | 기준 | 판정 | 메모 |
|---|---|---|---|
| Obsidian 열람 | Vault가 정상적으로 열림 |  |  |
| 그래프 표시 | 주요 hub와 링크망이 비정상적으로 고립되지 않음 |  |  |
| 소비 스킬 읽기 | 집필/로어북/번역 작업에서 표제어 참조 가능 |  |  |
| 보조 문서 | manifest에 등록된 필수 aux_docs가 존재 |  |  |
| 인덱스 | 전체 표제어 목록 또는 인덱스가 최신 |  |  |
| handoff 기록 | 다음 작업자가 상태를 이해할 수 있음 |  |  |

## 핸드오프 체크박스

```markdown
## Handoff Checklist

- [ ] 자동 단 FAIL 0건
- [ ] 수동 샘플 단 FAIL 0건
- [ ] WARN 항목의 사유와 후속 조치 기록
- [ ] LLM 완성률 95% 이상
- [ ] 필수 보조 문서 존재
- [ ] manifest `ready_for_consumption` 값 확인
- [ ] 발간 LOREBOOK 또는 공식 본문과 충돌 없음
- [ ] 다음 소비 작업과 버전 번호 공유

handoff_status: ready_for_consumption / ready_with_warnings / not_ready
handoff_date:
handoff_owner:
handoff_note:
```

## 운영 판단

개인 연습용 Vault는 WARN이나 일부 미완성을 안고도 학습 목적으로 사용할 수 있다. 그러나 소비 스킬에 넘기는 운영용 Vault는 이 체크리스트를 통과해야 한다. 기준을 느슨하게 두면 시간은 앞에서 아끼고, 오류는 뒤에서 몇 배로 갚는다.
