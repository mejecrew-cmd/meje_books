# 제6편. 실무편 A. 프로젝트 프로파일 템플릿

프로젝트 프로파일은 라이브러링의 첫 고정점이다. 1차 추출 전에 이 파일을 만들어 두면, 이후 2차 통합의 세계관축, 4차 서술의 톤, 검증의 기준이 흔들리지 않는다.

아래 양식은 그대로 복사해 `{프로젝트명}_profile.md`로 사용한다.

```markdown
# {프로젝트명} - 라이브러링 프로젝트 프로파일

작성일: {YYYY-MM-DD}
작성자: {이름 또는 팀}
버전: v0.1
상태: draft / active / locked

## 1. 기본 메타

- 프로젝트명:
- IP 유형: 소설 / 웹소설 / 웹툰 / 게임 / 룰북 / 영상 / 혼합
- 라이브러링 목적: 신규 구축 / 기존 Vault 갱신 / 번역 준비 / 로어북 제작 / B2B 공유
- 입력 디렉토리:
- 입력 문서 수:
- 입력 총분량 추정:
- 목표 표제어 수 범위: 1,000~2,000 / 기타
- 목표 완료일:

## 2. 입력 문서군
```

## 사용 메모

프로파일은 한 번 쓰고 버리는 문서가 아니다. 세계관축을 바꾸거나 입력 문서군을 추가하거나 4차 톤 기준을 바꾸면, 이 파일의 결정 로그에 남긴다. 그래야 3차 manifest와 10장 검증에서 "왜 이 상태가 되었는가"를 추적할 수 있다.
