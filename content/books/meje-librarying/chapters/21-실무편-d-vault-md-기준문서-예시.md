# 제6편. 실무편 D. Vault `.md` 기준 문서 예시

이 실무편은 3차 골격 빌드가 만들어 내는 기본 `.md` 파일의 모양을 보여 준다. 실제 프로젝트에서는 2차 CSV 한 행이 이 구조의 파일 하나로 변환된다.

## 표제어 파일 예시

파일명: `무문자.md`

```markdown
---
title: 무문자
aliases:
  - 무늬 없는 자
  - 장부 밖의 사람
category: 인물
worldbuilding_axis: 기억
english: Munmunja (the unwoven)
romanization: Munmunja
version: 현행
tags:
  - category/인물
  - axis/기억
  - version/현행
sources:
  - 가공본문 1문단
related:
  - "[[맨 기억]]"
  - "[[씨족 장부]]"
  - "[[고쳐 짜기]]"
  - "[[누비]]"
---

**무문자는 무늬를 읽거나 짜지 못하지만 맨 기억으로 씨족의 일을 보존하는 인물 또는 계층이다.**

무문자는 [[누비]]와 [[씨족 장부]]가 지식을 독점하는 사회에서 기록 바깥의 기억을 담당한다. [[고쳐 짜기]]가 공식 기록을 바꿀 때도 무문자의 [[맨 기억]]은 지워진 이름을 되살리는 근거가 된다.

**번역**: Munmunja (the unwoven). 첫 등장 시 괄호 해설을 붙이고 이후 Munmunja로 쓴다. 단순히 "illiterate"로 옮기면 무늬를 읽지 못한다는 세계관 고유의 함의가 사라지므로 금지한다.

%% LLM %%

무문자는 이 세계가 기록을 천에 짜 넣는다는 전제를 가장 조용히 거스르는 존재다. 그는 무늬를 읽지 못하고, 씨족 장부의 법을 온전히 따르지도 못하지만, 바로 그 결핍 때문에 고쳐 짜기로 바뀐 기록의 빈틈을 알아차린다. 모두가 새로 짜인 누비를 공식 기억으로 받아들일 때, 무문자의 맨 기억은 장부 바깥에 남은 이름을 붙든다. 그래서 무문자는 무능력자가 아니라, 기록 체계가 스스로 지운 것을 되묻는 사람이다.

%% /LLM %%

%% 메모 %%

- 4차 샘플 문체 기준: 기록/기억/배제의 긴장을 유지한다.
- 번역 시 illiterate 금지.
- 향후 로어북에서는 "기록 바깥의 증인" 계열 제목으로 확장 가능.

%% /메모 %%
```

## Stub 파일 예시

파일명: `무늬 없는 자.md`

```markdown
---
title: 무늬 없는 자
redirect: 무문자
type: stub
tags:
  - stub
---

[[무문자]] 참조.
```

## 구조 판정

| 영역 | 필수 여부 | 검증 방식 |
|---|---|---|
| frontmatter | 필수 | YAML 파싱 PASS |
| title | 필수 | 대표키워드와 일치 |
| aliases | 조건부 | 별칭이 있을 때 배열로 존재 |
| category | 필수 | 9분류 허용값 |
| worldbuilding_axis | 필수 | 프로파일 허용값 |
| related | 조건부 | 대상 표제어 또는 stub 존재 |
| 정의 | 필수 | 첫 굵은 문단 존재 |
| 번역 | 조건부 | 영문명/번역노트가 있을 때 존재 |
| `%% LLM %%` | 필수 | 4차 전에는 빈 블록 허용, 핸드오프 전 95% 이상 채움 |
| `%% 메모 %%` | 필수 | 작업 메모 보존 |

## 멱등 업데이트 규칙

- 재빌드 때 갱신: frontmatter, 정의, 상세설명, 번역노트.
- 재빌드 때 보존: `%% LLM %%`, `%% 메모 %%`.
- 표제어명이 바뀌면 기존 파일의 LLM/메모 이식 여부를 수동 검토한다.
