# 第28篇 商业模式设计框架——从历史中寻找我的商业模式

**核心问题：** 如何把至今读过的全部历史应用到自己的内容中？


## 用一句话概括本书想表达的内容

> **新的商业模式不是被发明出来的，而是被迁移过来的。**

在前27篇中，我们追溯了游戏、音乐、影像和特殊领域的商业模式历史，并发现了一个共同规律：成功的商业模式总是**落在既有的付费习惯空间里**。失败的商业模式要么试图进入一个不存在付费空间的地方，要么虽然选对了空间，却已有更强大的先行者占据那里。

现在，我们需要一套工具，把本书的逻辑应用到自己的内容上。本章就是这套工具。


## 工具1：付费习惯空间探索工作表

这个过程用来寻找自己的内容应该落在哪一种付费习惯空间。请依次回答以下五个问题。


**问题1：消费我的内容的人，目前在哪里消费这类内容？**

如果“我的内容”还不存在，人们会在哪里接触最相似的内容？相似内容当前的消费空间，就是判断付费习惯空间的线索。

示例：
- 想创作网络漫画 → 人们目前在哪里阅读相似的网络漫画？（Naver Webtoon免费／KakaoPage付费）
- 想创建烹饪视频频道 → 相似内容目前是在YouTube（免费），还是在Netflix纪录片（付费订阅）中？
- 想开发冥想应用 → Calm／Headspace已经存在按月付费的订阅空间
- 想开发AI写作工具 → 人们目前会订阅ChatGPT付费版（每月20美元），或使用Hancom Office
- 想发行独立游戏 → 人们已经习惯在Steam以15～25美元一次性购买独立游戏

**问题2：在那个空间里，金钱目前如何流动？**

是免费还是付费？如果付费，采用什么方式：一次性购买、订阅、应用内购买，还是观看广告？

这个问题的目的，是区分自己究竟要“创造一种付费习惯（非常困难）”，还是要“落在既有的付费习惯空间中（相对容易）”。

如果进入一个完全没有付费习惯的空间，并尝试第一次向用户收费，那就是在创造付费习惯。这并非不可能，但需要更多时间和成本。更现实的做法，是像YouTube在2007年刚出现时那样，先通过免费平台聚集受众，再进行商业化。

**问题3：我的内容解决了既有空间中的哪一种不便？**

如果既有的付费习惯空间已经存在，消费者在那里经历的不便，就是你的切入机会。

Blockbuster的不便是逾期费，Netflix取消了逾期费。
有线电视的不便是把用户不看的频道也捆绑销售，OTT让用户只订阅自己想看的内容。
音乐CD的不便是必须购买整张专辑，iTunes则让用户能够按单曲购买。

更多示例：
- 无限流媒体出现前，听音乐的不便：被迫购买整张CD专辑 → iTunes通过单曲销售解决
- 应用付费订阅的不便：不用的月份也要付费 → 通过“年度订阅预付折扣”解决（用户：固定费用可以预测；企业：提前获得现金）

## 工具2：商业模式类型选择决策树

弄清付费习惯空间后，下一步是选择哪一种商业模式。本书出现过的商业模式可以沿三个轴进行分类。


**轴1：重复性（Recurring vs. One-time）**

我的内容消费一次就结束，还是会产生重复消费？

- **重复消费型：** 订阅、月费、季票、应用内消耗品 → 适合订阅商业模式
- **一次性消费型：** 单行本、电影票、软件许可证 → 适合一次性销售商业模式
- **混合型：** 内容本身只消费一次，但属于一个系列 → 可考虑捆绑销售或季度订阅

我的内容的重复性：___________________________________________


**轴2：定制化（Customized vs. Standard）**

我的内容会因人而异，还是把相同内容大规模提供给所有人？

- **高度定制化：** 私人辅导、一对一咨询、定制报告 → 适合高价个性化服务商业模式
- **标准化：** 向许多人提供相同内容 → 适合低价大众订阅商业模式
- **平台：** 连接生产者与消费者 → 市场平台商业模式、平台佣金商业模式

我的内容的定制化程度：___________________________________________


**轴3：核心付费动机（Pleasure vs. Pain）**

客户为我的内容付费，是为了获得快乐，还是为了解决问题？

- **快乐（Pleasure）驱动：** 游戏、娱乐、爱好 → 存在价格阻力；充分的免费体验非常重要
- **解决问题（Pain）驱动：** 教育、医疗、法律、税务 → 价格阻力较低；对专业能力的信任非常重要
- **消除焦虑（Fear）驱动：** 算命、保险、安全 → 重复购买倾向较强；长期关系非常重要

我的内容的核心付费动机：___________________________________________


这三个轴的组合会指向最佳商业模式类型。

- **重复 × 标准化 × 快乐：** 订阅＋免费体验（Spotify模式）
- **重复 × 标准化 × 消除焦虑：** 订阅＋自动续订（保险、算命应用）
- **重复 × 高度定制化 × 解决问题：** 高价订阅＋一对一服务
- **一次性 × 标准化 × 快乐：** 单次购买＋系列捆绑（专辑、电影）
- **一次性 × 高度定制化 × 解决问题：** 按项目收费
- **混合 × 标准化 × 快乐：** 等待即可免费＋付费加速（KakaoPage模式）

**实际商业模式映射案例：**
- Spotify：重复 | 标准化 | 快乐 → 订阅＋免费体验
- KakaoPage：混合 | 标准化 | 快乐 → “等待即可免费”＋付费加速
- Calm：重复 | 标准化 | 消除焦虑 → 年度订阅＋自动续订
- Class101：一次性 | 标准化 | 快乐 → 单次购买＋系列捆绑
- Substack：重复 | 标准化 | 解决问题 → 每月5～10美元订阅＋作者策展


## 工具3：失败模式检查清单

本书讨论的失败案例可以归纳为七种共同模式。请检查自己的商业模式计划中是否存在这些模式。


**失败模式1：在没有付费空间的情况下启动收费**

在免费消费习惯已经牢固形成的空间里，没有足够的用户基础就尝试收费。

信号：“我们的服务目前免费，但只要推出高级方案，人们就会升级。”
案例：CNN+上线30天后关闭。
补充案例：Soribada在2004年启动收费。在免费共享习惯已经根深蒂固的空间里，它没有提供足够的替代价值便尝试转为付费，导致用户流失。

检查：我的免费版本是否已经拥有数十万名忠实用户？如果没有，收费还为时过早。


**失败模式2：以付费服务进入已被先行者占据的免费空间**

试图以付费服务进入一个已经由免费先行者支配的空间。

信号：“我们要做一个类似TikTok、但更好的短视频订阅服务。”
案例：YouTube与TikTok已经免费占领智能手机短视频空间后，Quibi以付费模式进入该空间。
补充案例：音乐视频流媒体VEVO在2009年进入市场时，YouTube已经占领免费音乐视频空间，因此很难实现独立商业化。

检查：我的空间中是否已有免费的先行者？如果有，我提供的附加价值是否足以跨过付费门槛？


**失败模式3：无法留住供应者的平台**

平台不给内容供应者提供收益，却只想聚集消费者。

信号：“只要创作者自由上传，内容就会积累，并吸引用户。”
案例：Vine没有向创作者分成，明星创作者因此转向YouTube。
补充案例：Cyworld的橡果没有给创作者——装饰迷你主页的人——带来收益；装饰好的内容又无法移出平台，离开平台就意味着放弃全部内容。

检查：供应者——创作者、作者或卖家——能否在我的平台赚钱？如果不能，他们就会离开。


**失败模式4：试图用技术改变习惯**

即使技术卓越，如果落点距离既有付费习惯太远，仍会失败。

信号：“我们的技术比现有技术好十倍。人们用上一段时间后就会付费。”
案例：MySpace Music拥有技术，却没能让用户形成付费听音乐的习惯。Jawbone曾是可穿戴技术的先驱，但产品质量问题与召回损害了信任，再加上与Fitbit的竞争和资金困难，最终于2017年清算。

检查：无论我的技术多么优秀，只要没有落在付费习惯空间里，就很难实现商业化。


**失败模式5：依赖单一IP**

收入过度集中在一个内容或IP上。

信号：“只要拥有［特定IP］，我们就足够了。”
案例：BTS宣布因服兵役暂停团体活动当天，HYBE市值下跌约2万亿韩元。Tidal的创始艺人支持减弱后，订阅用户开始流失。
补充案例：Warner Bros.在Harry Potter早期过度依赖单一原作IP的院线电影收益。直到原作完结后，才迫切需要扩展战略来维持IP价值。

检查：如果单一内容或IP贡献了总收入的50%以上，就是危险信号。


**失败模式6：价格超过付费习惯门槛**

无论服务多好，只要价格大幅超过既有付费习惯的金额，就很难实现转化。

信号：“考虑到我们服务的价值，每月50美元很合理。”
案例：3DO以699美元上市，远超当时200～300美元的游戏主机付费习惯，达到其两倍以上，最终失败。门槛在相反方向也存在：MoviePass以每月9.95美元、低于成本的超低价格进入市场，因而无法持续。价格既不能超过习惯门槛，也必须高于可持续成本。

检查：我的服务价格是否超过客户目前在相似类别中支出金额的两倍？如果超过，就需要一个非常有力的理由。


**失败模式7：过度扩张稀释原有价值**

成功后向过多领域扩张，导致原有粉丝群流失。

信号：“我们有IP，所以什么都能做。”
案例：有关“MCU疲劳”的讨论、Disney+过量制作Star Wars剧集、MCN因广告单价下降和内容质量降低而崩溃。
补充案例：Starbucks在2008年危机前过度扩张，门店数量增长过快，稀释了“特别的咖啡体验”这一核心价值。公司最终关闭600多家门店并重整品牌。

检查：当前扩张计划是在强化原有粉丝群的付费动机，还是在稀释它？


## 工具4：可迁移性评估矩阵

评估自己的内容能否自然落在既有的付费习惯空间里。每个项目按1～5分评分。

- **原有习惯空间的明确度**（得分：___）：“喜欢XX的人”可以直接成为我的第一批客户
- **迁移距离**（得分：___）：从既有习惯到我的内容，所需行为变化最小
- **价格熟悉度**（得分：___）：既有相似服务与我的价格区间相近
- **先行者的弱点**（得分：___）：既有先行者尚未解决的不便是否明确
- **重复消费结构**（得分：___）：开始消费后，是否有继续消费的理由
- **供应者收益结构**（得分：___）：内容供应者能否赚钱（适用于平台）

总分 **24～30分：** 很可能自然落在付费习惯空间中
总分 **15～23分：** 部分领域需要加强
总分 **6～14分：** 需要重新审视商业模式结构

**评分示例——KakaoPage引入“等待即可免费”商业模式时：**
- 原有习惯空间的明确度：5分（漫画租赁店读者构成了明确的原有习惯空间）
- 迁移距离：4分（虽然需要使用智能手机应用这一新行为，但门槛并不高）
- 价格熟悉度：5分（漫画店租借100～200韩元 → 每话约100韩元的虚拟币支付，价格区间相同）
- 先行者的弱点：4分（Naver Webtoon免费模式的缺点：无法一次读完已完结作品）
- 重复消费结构：5分（每天消耗虚拟币或等待24小时）
- 供应者收益结构：4分（平台向作者支付稿费）
- **总分27分** → 可以自然落地

**1．谁付钱？**

内容的最终消费者与付钱的人可能不同。儿童游戏由父母付费；B2B软件由企业而非用户付费；企业广告由广告主而非消费者付费。

**2．付钱买什么？**

不能只说是为了“使用我的应用”，而要明确究竟在为什么价值付费。

是便利（节省时间）、稀缺性（限量版）、身份（展示）、归属感（粉丝俱乐部）、安心（消除焦虑）、提升能力（教育），还是娱乐（快乐）？

**3．为什么现在付钱？**

引发付费的心理契机（trigger）是什么？如果没有这个契机，客户就会说“以后再买”，而不会发生转化。

TikTok爆红 → 今天播放 → 今天成为粉丝 → 今天产生购买专辑的契机
考试合格后 → 为了纪念 → 购买高价课程

**4．什么时候付钱？**

付费时机会决定商业模式结构。

预付（传统电影院、购买游戏主机游戏）vs. 后付（转为付费方案后）vs. 重复付费（订阅）vs. 条件付费（按效果收费）

**5．如何付钱？**

支付方式会产生摩擦。信用卡、应用内支付、预付充值、后付、自动续订等方式的摩擦越少，转化率就越高。

Amazon 1-Click消除购买摩擦后，购买率大幅上升。从Soribada转向iTunes时，0.99美元带来的“单次购买摩擦”已经足够低。

我的商业模式的支付方式与摩擦：___________________________________________


**6．在哪里付钱？**

物理与数字环境可以促进或抑制付费。便利店里的角色扭蛋机、演唱会现场销售的周边商品、应用内支付与网页支付，分别创造不同的付费情境。

在街机上投入硬币这一物理行为，把“正在花钱的感觉”降到最低。Netflix自动续订则让用户推迟“决定取消”的时刻。

**7．付多少钱？**

价格距离既有习惯空间的参照点越远，转化就越困难。Spotify每月10美元，比一张CD的15美元更便宜。KakaoPage每话约100韩元的虚拟币支付，与漫画租赁店的租金处在相同价位。

是单次还是重复？如果重复，频率是多少？这会决定LTV（客户终身价值）。

一次购买后，距离下一次购买要多久？缩短这个间隔，是重复收入的核心。

## 新内容商业模式策划书模板

整合前面填写的工作表和八个问题的答案，完成商业模式策划书的核心部分。


**A部分：定义习惯空间**

```
原有习惯空间：［我的客户原本花钱的空间］
原有习惯空间的核心付费模式：［如何、多少、何时］
我要落下的空间：［既有空间与我的内容的交叉点］
```

［示例——独立游戏开发者］
原有习惯空间：在Steam购买独立游戏（15～25美元一次性购买）
原有习惯空间的核心付费模式：在发行日或折扣季一次性购买
我要落下的空间：Steam独立游戏＋Kickstarter早鸟预购


**B部分：迁移逻辑**

```
既有空间的不便：［哪里不便］
我的内容如何解决：［如何做到更方便、更便宜或更好］
这次迁移自然的原因：［客户认为“这样更好”的理由］
```

## 本书没有说的事

最后，有一件事必须坦率说明。

本书通过历史案例展示了商业模式的规律，但本书提供的是**分析工具**，而不是**成功保证**。
