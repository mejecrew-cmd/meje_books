# 第12章．输出至五种媒介：Obsidian、TRPG、剧本、动画与网络小说

到上一章为止，一个人物已经抵达能够充分活着、行动的阶段。从种子到五幕变迁，十二个阶段中已有十一阶段完成。角色的内部、外部乃至时间轴，都已完成对齐。

本章是将这个人物展开到五种媒介的阶段。同一角色会在 Obsidian 统一资料库中呈现一种样貌，在 TRPG NPC 表中呈现另一种样貌，在影视剧本的一页角色圣经中呈现又一种样貌，在动画、漫画的视觉套件中呈现另一种样貌，在网络小说的压缩表中则呈现另一种样貌。

媒介改变时，角色的衣服会改变。角色的本质仍留在原处。

> **聚焦・《巫师》的杰洛特** 利维亚的杰洛特诞生于安杰伊・萨普科夫斯基的小说，后来进入 CD Projekt 的游戏，又成为 Netflix 的剧集。每种媒介的衣服不同：小说突出冷嘲的内心独白，游戏突出选择和剑术，剧集则突出演员的表情。但“猎杀怪物却知道真正怪物是人类，自称中立却最终站在弱者一边的变异猎人”这一内核，在任何媒介中都未改变。他最清晰地说明了本章的命题：完整构建一次，按媒介输出。

## 完整构建只做一次，按媒介输出

先确认这个立场。

不要为每种媒介重新塑造角色。完整构建只做一次。先构建从种子到五幕变迁的全部产出，再将这些产出转换为五种媒介。媒介改变，角色的本质也不会改变。

这种分离是本阶段的核心。若把媒介转换当成完整构建的一部分，角色会在每种媒介中逐渐偏移。在系列作品、IP 作品中，书中的角色、游戏中的角色、动画中的角色将各自拥有不同质地。读者和玩家必然会察觉到这细微偏移。

将媒介转换单独置于输出阶段，角色的本质便能得到保存。同一个人在五种媒介中活着、行动；只是各媒介所强调的部分不同。

下面依次查看五种媒介的输出格式。

## 第一种媒介：用于 Obsidian 统一资料库

首先要看的媒介是统一资料库的标准表。它是放入长篇小说、世界观 IP 统一资料库的格式。

这一格式是一张将一个角色全部资料汇集于一处的统一表。它将成为转换到其他媒介时唯一的事实来源。

Obsidian：一款以链接串联 Markdown 笔记的个人知识管理应用。笔记之间可以用 `[[链接]]` 将资料编织成蜘蛛网，因此常被用作汇集角色与世界观资料的“统一资料库”。重点不在于一定要使用这款应用，而在于把所有资料汇集到单一事实来源的思路。

格式骨架：

```yaml
---
title: (角色名)
codename: (工作中的临时代号)
created: YYYY-MM-DD
updated: YYYY-MM-DD
phase_completed: 11
type: character
project: (项目名)
status: ready_for_consumption
character_tier: protagonist | major | supporting | minor
length_target: (篇幅标准)
output_format: obsidian_master

# 镜像
mbti: 
mbti_main_function: 
hogwarts: 
big_five: 
enneagram: 

# 身份化
gender: 
age: 
occupation: 
appearance_signature: 
body_anchor: 

# 核心压缩
identity_word: "(单词身份)"
flaw_primary: 
trauma_type: big_t | small_t
want: 
need: 
lie: 
arc: positive | flat | negative

# 冲突
core_conflict: 
stakes: [...]
fun_engines: [...]

# 关系

# 调性
reference_authors: { foreign:, korean:, tone_oneliner: }

# 来源

# Vault

```

这二十五个 frontmatter 字段将角色的全部质地压缩在一处；正文再展开那种质地所穿的散文衣服。

```markdown

[优势・副作用 + 反差魅力]
```

这一格式会成为统一资料库中的一张角色页。M 的 Obsidian 表便以此格式构成。

转换为 Obsidian 格式，是与 AI 工作伙伴协作时优势格外明显的部分。输入十二个阶段的全部产出，即可在短时间内建立二十五个 frontmatter 字段与正文。制作人负责验收合格线。

## 第二种媒介：TRPG、游戏 NPC 表

TRPG（桌面角色扮演游戏）、游戏 NPC 用表，具有不同于统一表的质地。它优先采用第三人称客观叙述而非第一人称台词，尽量减少情感描写，并包含能力值区块。

角色表的格式本身就揭示了媒介的哲学。战斗为中心的游戏，其攻击力与体力栏位较大；以恐怖为中心的游戏如《克苏鲁的呼唤》，则另设“理智（SAN）值”栏位。建立什么栏位、留空什么栏位，正是在说“这个媒介中人物的什么最重要”。同一个人物，也会因媒介而装入不同的栏位。

核心一句：**记录的质地。** 它是记录，而不是小说。

frontmatter：

```yaml
---
title: (角色名)
output_format: npc_pc_history
character_tier: 
type: npc | pc

# D&D 5e 格式（可选）
abilities:
 STR: __ # 力量
 DEX: __ # 敏捷
 CON: __ # 体质
 INT: __ # 智力
 WIS: __ # 感知（洞察・直觉）
 CHA: __ # 魅力（存在感・口才）
alignment: lawful_good | true_neutral | etc.

# Personality 四栏
personality_trait: 
ideal: 
bond: 
flaw: 

# （以下为 Obsidian 基础字段）
---
```

正文格式：

```markdown
# (角色名)

> [一句：职业・身份・标志]

## 外貌
[一段。第三人称客观叙述。整合身体锚点和外貌一要素]

## 背景（Background）
[三段。第三人称客观叙述。出身・前史・当前职业。无情感评判]

## 能力（Abilities）
- 职业技能：[3～5项]
- 标志能力：[一项 + 优势・副作用]
- 身体能力值：STR/DEX/CON/INT/WIS/CHA

## 动机（Motivation）
- 即时目标：（Want）
- 长期目标：（Need）
- 秘密：[只有角色知道的一行]

## 关系
- 同盟（Ally）： 
- 敌对（Antagonist）： 
- 微妙关系（Complicated）： 

## 行动模式（Behavior）
- 平时： 
- 受威胁时： 
- 在信任的人面前： 

## 台词（Quotes，3～5句）

## GM／制作人备注
- [禁止公开的信息]
- [隐藏的秘密]
```

即使是主角级，篇幅也控制在四千字以内，尽量减少情感和内心描写。

GM（游戏主持人）：在桌面角色扮演游戏中推进故事、操控世界和 NPC 的主持者兼裁判，也叫地下城主（DM）。NPC 表中的“GM 备注”是让主持人记录如何扮演、处理该人物，以及还有哪些信息不应让玩家知道的栏位。

```yaml
## 动机
- 即时目标：追查在某具遗体手上发现的结婚戒指属于谁。

- 秘密：她把戒指放进包里，同时违反了程序。

- 同盟：同事验尸官（三十年同行）
- 敌对：检察官竞争者（察觉程序违规）
```

这就是角色进入游戏 NPC 时所穿的衣服。它比 Obsidian 格式更压缩、更客观。游戏主持人（或游戏设计师）会摊开这张表来运用角色。

NPC 格式转换是与 AI 工作伙伴协作后能显著加速的部分。将 Obsidian 表作为输入，便能在短时间内转换为 NPC 格式。

### TRPG 辅助工具：GURPS 优势与劣势点数

除了 D&D 5e，还有另一种常用于 TRPG 角色表的格式：GURPS 的优势与劣势点数系统。

**背景。** GURPS（Generic Universal RolePlaying System）是 Steve Jackson Games 于 1986 年出版的通用 TRPG 系统。其特点是在能力值之外，以点数让**优势（Advantages）与劣势（Disadvantages）保持平衡**来构建角色。

**导入宗旨。** 如果 D&D 5e 的六项能力值加上四个 Personality 栏位是在把握角色的外部形状，GURPS 的优势与劣势则用点数格式整理缺点的两面性（第 7 阶段）。它能用数字验证一个角色的能力和缺点是否平衡。

**预期效果。** 只有强大能力的角色（优势 +20pt、劣势 0pt）是扁平的。优势和劣势以相近点数总和保持平衡的角色（优势 +15pt、劣势 -12pt）更立体。GURPS 格式会用数字显示这种平衡。

```markdown
优势（5～10项，合计点数）：
- __（+__pt）
合计：+___pt

劣势（3～7项，合计点数）：
- __（-__pt）
合计：-___pt

净点数：___（优势 - 劣势，建议在 ±10pt 内）
```

试着套用到 M。

```
M 的优势：
- 验尸精确度 +5pt（由遗体手指厚茧推定职业）
- 三十年经历 +3pt
- 同事的信任 +2pt
- 洞察力（WIS 17） +4pt
合计：+14pt

M 的劣势：
- 回避私人领域 -4pt
- 婴儿爽身粉触发物 -2pt
- 母亲缺席的创伤 -5pt
- 社会野心薄弱 -2pt
合计：-13pt

净点数：+1pt，平衡
```

优势略高于劣势的 +1pt，是立体的平衡。并非所有角色都必须套用 GURPS，但当 TRPG 游戏主持人想用数字确认角色平衡时，它是实用的辅助工具。

## 第三种媒介：影视剧本的一页角色圣经

编剧所用的格式压缩在一页内，约 1,800 字。一页中压缩角色的全部质地。

Blizzard 的《守望先锋》为每个角色设有“角色圣经”，把外貌、性格、来历、关系及说话方式整理成文档。活泼的猎空、沉默的源氏、玩世不恭的牛仔卡西迪（原麦克雷）等，每个人物的质地都被清楚记录，因此即使出现新皮肤、短篇动画或与其他游戏的联动，人物也不会走样。一份写得好的圣经，能守住无数衍生内容的一致性。

```markdown
# （角色名）：Character Bible

**Logline**: [一句：谁在何种情境中想要什么]

**Age / Occupation / Status**: __ / __ / __

**Want（外部目标）**: [一句]
**Need（内部目标）**: [一句]
**Lie（所相信的谎言）**: [一句]
**Wound（仍在流血的伤）**: [一句]

**Dominant Trait**: [一个词]
**Defining Quirk**: [一句，telling detail]

**Arc Direction**: Positive / Flat / Negative

**Story Spine**（Pixar 七句）：
- Once upon a time:
- Every day:
- Until one day:
- Because of that:
- Because of that:
- Until finally:
- And ever since:

**Voice**:
- 说话方式一句（称呼・语调）：
- 第一行：
"___"

**Key Relationships**（3～5）：
- __（Ally / Mentor / Antagonist / Foil）：一句

**五项诊断测试**：Bechdel __ / Mako Mori __ / Sexy Lamp __ / Walks-into-a-room __ / One-Line __
```

试着把 J（作家型）输出为一页剧本圣经。

```markdown
# J：Character Bible

**Logline**: 一名三十多岁作家的某个时期：她渴望成为畅销书作家，却努力不失去写作行为本身的乐趣。

**Age / Occupation / Status**: 35岁 / 作家，电影导演与韩国文学双主修 / 未婚

**Want**: 被认可为畅销书作家。
**Need**: 接受作品的价值和自身价值可以分离。
**Lie**: "只有成果有趣，才有价值。"
**Wound**: 每次写作前都把手洗到发红的强迫，每天重新流血。

**Dominant Trait**: 理想主义者。
**Defining Quirk**: 总是带着黄色 Oxford 便签本，却从不使用。

**Arc Direction**: Flat。她从第一幕便知道 Need，但外部世界将她拉向 Want。改变的不是 J，而是 J 周围的世界。

**Story Spine**:
- Once upon a time, J 在自我怀疑中每天写作。
- Every day, 她写作前都把手洗到发红。
- Until one day, 一名畅销书作家向她提出导师邀请。
- Because of that, J 第一次怀疑写作行为本身的意义。
- Because of that, J 知道导师的姿态与自己的 Lie 完全相反。
- Until finally, J 拒绝导师的邀请。
- And ever since, J 在同一个地方、以同一种姿态写作。改变的是导师的认知。

**Voice**:
- 说话方式：平静的第一人称，思辨和实务混杂，没有愤怒。
- 第一行："我从三十年前就知道，写作并不有趣。"

**Key Relationships**:
- 畅销书导师（Antagonist）：最强化 J 的 Lie 的人物，是 J 必须拒绝的阶段。
- 写作同伴（Confidant）：三十多岁时一同写作的同伴，彼此催逼截止日期。
- 出版社编辑（Foil）：比 J 更结果导向，每天动摇 J 的姿态。

**五项诊断测试**：Bechdel △ / Mako Mori ○ / Sexy Lamp ○ / Walks-into-a-room ○ / One-Line ○
```

J 的所有质地已压缩在 1,800 字内。编剧摊开这一页，便能编写一季的 J 台词。

这种格式转换也是与 AI 工作伙伴协作时可以快速处理的部分。

## 第四种媒介：动画、漫画视觉套件

动画、漫画媒介以视觉为先。格式的核心是五种视觉元素。

轮廓测试是角色设计的基本功。人物即使只成为漆黑的影子，也应让人认出是谁，例如米老鼠的圆耳朵、《龙珠》孙悟空竖起的头发、《宝可梦》皮卡丘的闪电尾巴。这也是游戏和动画公司从角色轮廓开始做出差异的原因。即使用文字描绘人物，也适合先决定“远看成影子也能认出的那一件事”。

```markdown
"两次便能保存。"

## 五情绪比例
M 的五情绪比例为：快乐 5%（几乎没有）、悲伤 35%（沉默的深度）、愤怒 5%（几乎没有）、厌恶 25%（回避私人领域）、恐惧 30%（在超市自助结账机前停住）。
```

五情绪比例打破平衡，集中于悲伤、厌恶和恐惧。比例捕捉到作品虽是复原叙事，底下却流着深沉悲伤的事实。动画作画团队可以摊开这一比例，设计表情和姿势。

## 第五种媒介：网络小说压缩表

网络小说是类型惯例很强的媒介，格式随类型而不同。

网络小说必须在第一话迅速让人物留下印象才能生存。《我独自升级》的成振宇从“世界上最弱的猎人”一句开始，立刻吸引读者；《全知读者视角》的金独子用“十年间只读完一部小说的普通上班族”一句刻下自己。媒介越没有余裕铺陈长篇说明，用一句话立起人物的压缩力就越是生存力。《华山归还》的青明也以“华山派传说高手在百年后回到衰败门派，成为年少弟子”一句，从第一话便抓住读者。

```markdown
# E

**回归轮次**：第二次。

**现世职业**：同样是受雇杀手，但为救七岁人质孩子而回归。
```

## 媒介之间的转换

同一个角色在五种媒介中活着、行动。先整理哪些被保存、哪些会改变。

故事脊线（Story Spine）是一种填入“很久很久以前……、每天……、直到有一天……、因此……、终于……、从那以后……”空格的短故事骨架。这是 Pixar 编剧常用的检查框架；把完成的角色放进这几句，人物的一生便一目了然。它适合作为输出至媒介前最后的检查工具。

媒介转换中，被保存和改变的内容是分开的。单词身份会被保存，但被强调部分的比例会改变；Want、Need、Lie 三行会被保存，但格式的压缩程度会改变。缺点的两面性会被保存，但细节的丰富度会改变；身体锚点会被保存，但视觉和听觉的明示程度会改变；五幕变迁的骨架会被保存，但一幕的长度会改变；创伤链条会被保存，但情感描写的外露程度会改变。

角色本质被保存的部分，与穿上媒介衣服的部分是分离的。本阶段的工作，是不改变本质而为角色穿上适合媒介的衣服。

## 谁做什么

这一阶段在与 AI 工作伙伴协作时最能显现优势。五种媒介的格式转换都适合自动化：以 Obsidian 表作为输入，其他四种媒介的格式便能在短时间内转换。制作人只验收各媒介的合格线。

各媒介的合格线：

```
- Obsidian：可在统一资料库中检索，与其他角色的链接准确
- TRPG NPC：游戏主持人可凭一页运用角色
- 一页剧本：编剧可凭一页编写一季台词
- 动画套件：作画团队只用五种视觉元素即可画出角色
- 网络小说压缩：类型读者可凭一页抓住角色魅力
```

五条合格线全都不同。预先决定每种媒介以什么作为合格线的姿态，正是本阶段的核心。
