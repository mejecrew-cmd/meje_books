# 第5章　单个词条的形态——标准模板与frontmatter

想象一条Wikipedia词条。无论是李舜臣、华城行宫还是壬辰倭乱，最上方是一行摘要，下面是导入段落，正文按`##`标题分节，最后是相关文档、外部链接和注释。转到下一词条，形式仍然相同。

LOREBOOK的词条格式是作者与读者之间的契约。无论翻开哪页，同类资料都在同一位置；从第1项到第1,500项，一行摘要、导入、正文、相关词条的次序不变。容器相同，资料之间的差异反而更清晰。这正是《Forgotten Realms》自1987年以来维持同一结构、《Final Fantasy Ultimania》从X（2001）到XVI（2024）反复使用同一跨页的原因。

## 单个`.md`文件的五部分

一个文件由五部分组成，对应不同的**阅读速度**。

frontmatter是读者看不见的YAML元信息，支撑搜索、部映射和多语言一致性。一行摘要以`>`开头，不超过100个韩文字符，让读者一秒确认身份。导入段落没有标题，为一至三段、200至600个韩文字符，让读者一分钟把握全貌。正文以自然的`##`标题展开，是停留约十分钟的部分。相关词条列出五至十个wikilink，是通往下一项的出口。五部分形成一秒、一分钟、十分钟与移动的阶梯。

## frontmatter——看不见的资料

```yaml
---
title: 神降仪式
aliases: [降神礼仪, Naerimgut]
category: 装置
worldbuilding_axis: 巫俗
english: Spirit Possession Ritual
romanization: Sinnaerim Uisik
version: 现行
hub_or_leaf: hub
lorebook_section: 02-§3-巫俗体系
tags:
  - category/装置
  - axis/巫俗
  - version/现行
  - lorebook/hub
  - lorebook_section/02-§3-巫俗体系
sources:
  - 规则书第3版 p.24
  - 短篇047
related:
  - "[[男巫]]"
  - "[[郁结]]"
  - "[[附身]]"
translation_notes:
  - 推荐音译: "Sinnaerim Uisik"
  - 英文说明: "spirit-descent ritual; the moment when a Mudang first hosts a deity"
---
```

`title`用于正文与索引，`aliases`辅助别名检索，`category`和`worldbuilding_axis`是双坐标，`english`和`romanization`用于外语版与首次并记，`version`阻止废弃项进入正文。LOREBOOK再加`hub_or_leaf`、`lorebook_section`和专用`tags`。多语言IP共有12个字段，否则为11个。准确元信息把搜索、映射和多语言发行移入自动化。

## 一行摘要——一秒的承诺

```markdown
> 男巫第一次接受神灵的仪式。巫堂一生中最沉重的时刻。
```

“本页讨论神降仪式”这样的元说明没有信息。好摘要会立即传达核心。六风格地区可写为“六个清晨：决定一座城市性格的六个地区”；黑孔可写为“本IP所有机制都从这一缕黑烟流出”。Waterdeep与Sharn的市场案例也把“在IP中的位置”和“读者首先应知道的事”压入一行。

## 导入段落——一分钟的全貌

```markdown
神降仪式是男巫第一次把神灵接入自身的仪式。结束后，巫堂成为神的通道。仪式持续数日，家人与邻里共同参加。

它是巫堂职业的开始，显示本IP的巫俗体系如何决定一个人的一生。若发生附身，他被承认为正式男巫；若失败，则不能再试。
```

导入需要三项：流畅的定义、在IP中的位置与其他词条的关系、读者为何应了解它。超过800个韩文字符会推迟正文，少于150则桥梁不足。

## 正文分节——自然标题

```markdown
## 仪式的进程
神降仪式持续数日或数周。...

## 在巫堂家的房屋中
仪式通常在巫堂家的房屋举行。...

## 仪式后的变化
成功完成仪式的巫堂... 
```

标题本身应说明时间、空间或变化。避免“信息”“详细说明”“概要”等通用元标签，清除TBD、WIP等临时标记，也避免报告式的“1. 环境影响”。

## 词条类别——卡片、标准、深层

**卡片型**为800至2,000个韩文字符、约1页，适合NPC、企业、工具和短事件，共约1,000至1,200项，主要是leaf。**标准型**为1,500至3,000字符、1.5至2页，适合地点、日常、文化，共200至300项。**深层型**为3,000至5,000字符、2至3页，适合城市、势力与核心法则，共100至150项，主要是hub。

自然比例为深层7%至10%、标准13%至20%、卡片70%至80%。引用次数可自动提出候选，但核心人物与配角的实际比重须由了解IP的人决定。

## 相关词条——通往下一项的桥

```markdown
## 相关词条
- [[男巫]]。仪式后的职业
- [[郁结]]。仪式所释放的对象
- [[附身]]。仪式结果产生的现象
- [[巫堂家的房屋]]。仪式常发生的地点
```

它把Librarying的`related`展开为五至十个链接，并附简短关系说明。1,500项可形成5,000至10,000条链接，LOREBOOK还可补充部内连接。关系说明让读者能够选择下一项。

## 编辑注记与页脚——不向读者显示的资料

```markdown
<!--
编辑注记:
- 原始来源: Vault [[神降仪式]]
- frontmatter同步日期: 2026-04-30
- 插画规格: 仪式场景、全彩、可用跨页
- 未确定设定: 仪式持续时间
- 写作问题: 检查附身描写是否过强
- 状态: ●
-->
```

HTML注释保存来源、同步日期、插画、未确定设定、审查问题和六阶段进度，在预览、网页与出版中隐藏。最后一行记录位置、版本和更新日期。

```markdown
*- 02_第2部_世界法则/03_巫俗体系/01_神降仪式 / [LOREBOOK名] v0.3 / 2026-05-01*
```

## 直接转换Librarying frontmatter

从13栏CSV机械映射：代表关键词→`title`，分类→`category`，世界观轴→`worldbuilding_axis`，别名→`aliases`，英文名→`english`，罗马字→`romanization`，版本→`version`，翻译注释→`translation_notes`。定义进入摘要与导入首句，详细说明进入导入，相关关键词进入`related`，来源进入`sources`，`IDX`不用。LOREBOOK再增加三个字段。

机械映射可在约五分钟生成1,500个frontmatter完整的文件，把人工一周的工作缩短到几分钟。

## 相同容器使差异更清晰

摘要、导入、正文、相关词条始终处于同一位置，读者无需重新学习结构，注意力会直接落到内容。神降仪式与人物Kim Mi-seon使用同样五部分时，仪式与人物、机制与人生、抽象与具体的差异会作为内容差异抵达读者。

读者能顺畅从一项移动到下一项，一半来自资料的趣味，一半来自格式的一致。标准模板不是限制作者自由，而是建立读者信任。
