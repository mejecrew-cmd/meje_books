# 第9章　短篇引用与多语言出版——收入正卷与排除在外的内容

设想书店的书架上摆着一本《Forgotten Realms Campaign Setting》（2001），旁边还有《Realms of Mystery》（1998）、《Realms of Infamy》（1994）等十多本费伦短篇集。两类书都是同一 IP 的资料，却是不同的书。一类是城市、人物与势力的目录，另一类讲述一个人物或一场事件。系列化封面让人一眼看出它们属于同一 IP，但书的身份与读者用途彼此分开。CD Projekt 的《The Witcher》也采用同样结构：《Compendium》（LOREBOOK）与萨普科夫斯基的短篇集分卷出版。定义一本书的，不只是书中收录了什么，也包括它**没有**收录什么。

有六类资料不进入 LOREBOOK 正卷：本 IP 的短篇全文；若 IP 有游戏规则，则包括规则数值；Librarying frontmatter 中标记为**未定**的词条；标记为**废弃**的词条；游戏元语言；以及内部整理资料所用的法则代码。排除这六类资料的理由，共同塑造了 LOREBOOK 的性质。

这里要引入 SSOT（Single Source of Truth，单一事实源）。它规定一项资料由哪里作出决定。若在 SSOT 层级中 LOREBOOK 低于规则书，规则数值就由规则书确定，LOREBOOK 只接收该决定。未定词条若进入正文，LOREBOOK 本身也会变成未定稿，因此须在 Librarying 阶段确定后再移入正文。废弃词条不出现在正文，可与废弃理由一起列入附录1术语表，或只在编辑注（HTML 注释）中留下痕迹。“玩家掷骰子”这样的游戏元语言和 PHYS-M01 这样的法则代码，也应进入规则书、附录或编辑注等其他区域。

六类资料中体量最大的是短篇正文。即使一个 IP 运营五年后积累了五十至一百篇短篇，全文也不进入 LOREBOOK。本章前半讨论原因及短篇的处理方式，后半讨论把正卷转移到其他语言的多语言出版。两个主题放在同一章，是因为它们都在决定**什么进入正卷，什么留在正卷之外**。

## 短篇另册出版的理由

假设一个 IP 有一百篇短篇。每篇平均7,000个韩文字符，总计70万字，约为1,000至1,200页。若 LOREBOOK 正卷约450页，合订后就是1,450页。B5精装本的书脊会超过80毫米，难以单手托举。

仅凭篇幅，分卷就很自然；更决定性的理由是读者的**用途**不同。

LOREBOOK 读者为了检索或通览 IP 资料而翻书。他们要回答“这个人物是谁”“这座城市在哪里”“这场事件发生在哪里”等问题，并希望在一两分钟内从一个页面取得资料。字典与指南两种用途共存于一卷。

短篇集读者则按阅读时间体验 IP 的故事。他们在一次阅读中完整读完一篇，在另一个时段再遇见下一篇。短篇集几乎没有字典用途，通读方式也与 LOREBOOK 不同：通览 LOREBOOK 是在词条之间移动，通读短篇集则是从头到尾读完一个故事。

若两种用途装订在一起，读者每次都要决定自己是在检索、通览 LOREBOOK，还是阅读短篇。这种决策负担会令整本书更难进入。分卷把负担拆开：正卷负责检索与通览，短篇别册负责故事阅读。读者只需选择符合当下用途的书。

短篇别册的出版规格宜与正卷成对。开本可与正卷同为B5，或采用便携的A5；一百篇×每篇平均6至10页，总计600至1,000页；装帧采用精装或软精装，并考虑体量与重量；封面延续正卷的系列设计。豪华版可用书盒同时收纳两卷。别册使用独立 ISBN，并在自身附录中提供按短篇、人物、视点和地点编制的索引。

## 短篇引用的三种形式——按深度区分的三个层级

短篇全文不进入正卷，但短篇引用会出现。它让读者在阅读 LOREBOOK 时短暂接触本 IP 的叙事资料，并按深度分为三种形式。

最深的一种是 *Pull Quote*（引用框）。当正文词条高度依赖短篇资料时，把短篇的一个段落以引用框置于词条中间。

```markdown
朴数巫金美善的一天从凌晨三点开始。她在家宅神坛前修整香条，
拨松一夜间凝固的香炉灰。这是金美善巫职一天中的第一项仪式。

> “那天的黎明也与其他黎明一样。香燃得更久了一些，一缕烟
> 触到神坛的天花板后散开。金美善在神坛脚下摊开手掌。
> 那一天，第一次有神触碰了那只手掌。”
> ——短篇集023《神坛的黎明》，首段

以神坛之香开始一天的景象，在本 IP 的巫俗家宅中普遍相似。
凌晨三点是一家人与神相遇的时间，也由此塑造了巫职的结构。
```

短篇的一段文字进入正文散文流，引用后的正文再承接并深入解释。标准做法是引用约50至200个韩文字符，注明出处（“短篇集XXX《标题》，段落位置”），并在引用后立即展开说明。每一部约出现5至10次。

较轻的形式是在**短引文后加上别册指引**。用一两行引文点明联系，再附上“全文见短篇别册§023”等说明。每一部约出现10至20次。

最轻的形式是**不直接引用，只指出语境**。例如：“金美善的降神仪式是短篇集023《神坛的黎明》的舞台。”正文不依赖短篇，只告诉想阅读故事的读者该去哪里。每一部约出现20至30次。

三种形式合计每一部约35至60次，一本十二部结构的 LOREBOOK 约有400至700处短篇引用。这一数量足以让 IP 的短篇资料自然流经正卷，又不会吞没正卷。

## 短篇与 LOREBOOK 发生冲突时

短篇与 LOREBOOK 可能出现资料冲突，例如人物履历在短篇中是A，在 LOREBOOK 中却是B。谁优先由 SSOT 层级决定。

标准顺序是：角色表最高，其次是经主审确认的短篇，再次是 LOREBOOK，最后是 Librarying Vault 的散文。根据 IP 的性质，规则书也可能排在更高位置。

这意味着 LOREBOOK 必须服从短篇中已经确定的事实。LOREBOOK 不能否定短篇；若正文资料与短篇冲突，应更新 LOREBOOK 以与短篇一致。

但短篇未涉及的领域适用反向原则。若短篇没有描写人物的童年，LOREBOOK 可以补足，并成为该资料的 SSOT。用一句话概括：**短篇写到的部分以短篇优先，短篇未写到的部分由 LOREBOOK 决定**。

正卷与别册之间还要遵守双向引用原则。正卷引用别册，别册也应引用正卷。短篇结束后的页边可写：“本篇《神坛的黎明》也见于 LOREBOOK §2-3 巫俗体系；金美善的详细资料见§9-1 巫俗人物群。”这样，读者便能从短篇前往 LOREBOOK，也能从 LOREBOOK 返回短篇。

《Forgotten Realms》短篇集与 LOREBOOK 三十年来的配合就是范例。自1988年第一本《Realms of Valor》以来，约二十本短篇集与 LOREBOOK 共同出版。短篇首页会标上 *Setting: Waterdeep, 1372 DR* 这样的坐标，LOREBOOK 的深水城词条则以引用框收录相关片段。读者可从故事转向城市的详细资料，也可从资料词条转向事件的完整故事。两本书承载同一 IP 的两种纹理，并彼此支撑。

## 多语言出版——建立在韩文正本之上

第二个主题，是在韩文 LOREBOOK 正本出版后将其转移到其他语言。

原则有三项：**韩文正本优先**、**多语言版采用独立周期**、**直接利用翻译注**。

韩文正本完成至v1.0后才开始多语言出版。若两项工作同时进行，韩文正本会受多语言版影响而偏移，多语言翻译又追不上正本更新，资料便分裂为两个分支。先完成正本，再为多语言版设置独立周期，才能保证一致性。

多语言出版的核心资料是 Librarying frontmatter 的 `translation_notes` 字段。Librarying 已预先填写每个词条的翻译方针，如建议音译、英文解释与建议脚注，多语言制作可直接接收这些资料。

虚构 IP“○○”中“降神仪式”的翻译注如下：

```yaml
translation_notes:
  - 建议_音译: "Sinnaenim Uisik"
  - 英文_解释: "spirit-descent ritual; the moment when a Mudang first hosts a deity"
  - 建议_首次出现加脚注: true
  - 无法简单翻译的原因: "Western 'shaman' is broader; Mudang is gender-specific"
```

这些资料在英文 LOREBOOK 中形成如下正文：

```markdown
The Sinnaenim Uisik¹ is the first ritual in which a Mudang receives
a deity. The ceremony lasts for several days, with family members
and neighbors present. ...

¹ Spirit-descent ritual. The Korean term Sinnaenim ("spirit-descent")
  Uisik ("ritual") refers specifically to the moment when a Mudang
  (Korean shaman) first hosts a deity. Western 'shaman' is broader;
  Mudang is gender-specific (often female).
```

音译名称出现在英文正文中，首次出现处添加脚注¹，预先写好的解释进入脚注。把韩文正本转成多语言版并非从白纸开始，而是接收 Librarying 已设计好的资料。

目的在于节省时间并保持一致。若从头处理，每个词条需30分钟至1小时，1,500个词条合计750至1,500小时；接收预备资料后，可缩短到约5至10小时的审校。一致性也来自上游：Librarying 阶段预先确定标准写法，各语言版本即可共同复用，避免因作品或译者而变化。

## 音译、意译与解释——一个称谓如何转移

多语言出版中，每个词条要在音译、意译和解释三条路径间作出选择。市场上的三个例子很有启发。

《哈利·波特》在同一作品内部就有明显分流。韩文版对“Hogwarts”“Muggle”“Quidditch”采用音译，对“Sorting Hat”按功能意译，对“Diagon Alley”则使用解释性的语音处理。接近专名或世界身份的资料倾向音译，指向功能的资料倾向意译。Librarying 翻译注若预先确定这些分支，韩文、日文和中文版便能保持一致。

托尔金的例子更明确。他亲自留下给译者的《Guide to the Names in The Lord of the Rings》，规定某些称谓如Frodo应音译，另一些如Bag End应按意义翻译。这份资料成为四十多种语言译本的 SSOT。LOREBOOK 的 Librarying 翻译注处在同一位置。

村上春树作品也有类似的一致性。《挪威的森林》英译本把“Watanabe”和“Gotanda”音译，却把“东京大学”解释为 University of Tokyo。人物名采用音译，机构和地名采用解释，规则始终一致。

三个例子说明，翻译实际上是**分类工作**。一个 IP 内的称谓按种类分为音译、意译和解释。分类良好，下一种语言可在已有版本上自然建立；分类失序，同一称谓会在不同版本中变形，使读者困惑。

## 多语言出版的三种形式

LOREBOOK 的多语言出版有三种形式。

最常见的是在韩文正卷之外另行出版英文版。两卷采用相同设计与规格配对，通常在韩文正本完成后约6至12个月推出。

较轻的形式是正卷仅用韩文，但在卷末部分附录（术语表、人名索引）并列英文写法。读者阅读韩文时也能取得英文形式。工作约需1至2个月，因为英文写法可直接来自 frontmatter 的 `english` 与 `romanization` 字段。

规模最大的是正卷多语言同步出版，所需时间至少为韩文正本的两倍。前两种是标准方案；同步出版只建议用于体量巨大的 IP。

## 正文标记原则

韩文正本的规则很简单：正文使用韩文；除必要的汉字或拉丁字母并记外，不直接插入外语正文；多语言写法只在 frontmatter、术语表和元数据层管理；其他语言版本使用独立周期制作。

在“降神仪式”的例子中，正文只出现韩文称谓。首次出现时可以在括号中简短并列汉字与罗马字，如“降神仪式（身降神 儀式, Sinnaenim Uisik）是……”。汉字只在相应 IP 确有需要时使用，罗马字遵循韩国国语罗马字表记法。Spirit Possession Ritual 这样的英文意译不进入韩文正文的首次出现处，只在制作英文版时进入英文正文。

收入什么、排除什么，决定了 LOREBOOK 的身份。当短篇全文、规则数值、未定词条、游戏元语言、法则代码和外语正文都有明确的外部去处，正卷所收资料的语调自然会显现。一卷书明确自己的领域，把领域之外的资料交给短篇别册、规则书或多语言别卷。书的分量不是由页数决定，而是由领域的清晰度决定。
