# 第5章　單個詞條的形態——標準模板與frontmatter

想象一條Wikipedia詞條。無論是李舜臣、華城行宮還是壬辰倭亂，最上方是一行摘要，下面是匯入段落，正文按`##`標題分節，最後是相關文件、外部連結和註釋。轉到下一詞條，形式仍然相同。

LOREBOOK的詞條格式是作者與讀者之間的契約。無論翻開哪頁，同類資料都在同一位置；從第1項到第1,500項，一行摘要、匯入、正文、相關詞條的次序不變。容器相同，資料之間的差異反而更清晰。這正是《Forgotten Realms》自1987年以來維持同一結構、《Final Fantasy Ultimania》從X（2001）到XVI（2024）反覆使用同一跨頁的原因。

## 單個`.md`檔案的五部分

一個檔案由五部分組成，對應不同的**閱讀速度**。

frontmatter是讀者看不見的YAML元資訊，支撐搜尋、部對映和多語言一致性。一行摘要以`>`開頭，不超過100個韓文字元，讓讀者一秒確認身份。匯入段落沒有標題，為一至三段、200至600個韓文字元，讓讀者一分鐘把握全貌。正文以自然的`##`標題展開，是停留約十分鐘的部分。相關詞條列出五至十個wikilink，是通往下一項的出口。五部分形成一秒、一分鐘、十分鐘與移動的階梯。

## frontmatter——看不見的資料

```yaml
---
title: 神降儀式
aliases: [降神禮儀, Naerimgut]
category: 裝置
worldbuilding_axis: 巫俗
english: Spirit Possession Ritual
romanization: Sinnaerim Uisik
version: 現行
hub_or_leaf: hub
lorebook_section: 02-§3-巫俗體系
tags:
  - category/裝置
  - axis/巫俗
  - version/現行
  - lorebook/hub
  - lorebook_section/02-§3-巫俗體系
sources:
  - 規則書第3版 p.24
  - 短篇047
related:
  - "[[男巫]]"
  - "[[鬱結]]"
  - "[[附身]]"
translation_notes:
  - 推薦音譯: "Sinnaerim Uisik"
  - 英文說明: "spirit-descent ritual; the moment when a Mudang first hosts a deity"
---
```

`title`用於正文與索引，`aliases`輔助別名檢索，`category`和`worldbuilding_axis`是雙座標，`english`和`romanization`用於外語版與首次並記，`version`阻止廢棄項進入正文。LOREBOOK再加`hub_or_leaf`、`lorebook_section`和專用`tags`。多語言IP共有12個欄位，否則為11個。準確元資訊把搜尋、對映和多語言發行移入自動化。

## 一行摘要——一秒的承諾

```markdown
> 男巫第一次接受神靈的儀式。巫堂一生中最沉重的時刻。
```

“本頁討論神降儀式”這樣的元說明沒有資訊。好摘要會立即傳達核心。六風格地區可寫為“六個清晨：決定一座城市性格的六個地區”；黑孔可寫為“本IP所有機制都從這一縷黑煙流出”。Waterdeep與Sharn的市場案例也把“在IP中的位置”和“讀者首先應知道的事”壓入一行。

## 匯入段落——一分鐘的全貌

```markdown
神降儀式是男巫第一次把神靈接入自身的儀式。結束後，巫堂成為神的通道。儀式持續數日，家人與鄰里共同參加。

它是巫堂職業的開始，顯示本IP的巫俗體系如何決定一個人的一生。若發生附身，他被承認為正式男巫；若失敗，則不能再試。
```

匯入需要三項：流暢的定義、在IP中的位置與其他詞條的關係、讀者為何應瞭解它。超過800個韓文字元會推遲正文，少於150則橋樑不足。

## 正文分節——自然標題

```markdown
## 儀式的程序
神降儀式持續數日或數週。...

## 在巫堂家的房屋中
儀式通常在巫堂家的房屋舉行。...

## 儀式後的變化
成功完成儀式的巫堂... 
```

標題本身應說明時間、空間或變化。避免“資訊”“詳細說明”“概要”等通用元標籤，清除TBD、WIP等臨時標記，也避免報告式的“1. 環境影響”。

## 詞條類別——卡片、標準、深層

**卡片型**為800至2,000個韓文字元、約1頁，適合NPC、企業、工具和短事件，共約1,000至1,200項，主要是leaf。**標準型**為1,500至3,000字元、1.5至2頁，適合地點、日常、文化，共200至300項。**深層型**為3,000至5,000字元、2至3頁，適合城市、勢力與核心法則，共100至150項，主要是hub。

自然比例為深層7%至10%、標準13%至20%、卡片70%至80%。引用次數可自動提出候選，但核心人物與配角的實際比重須由瞭解IP的人決定。

## 相關詞條——通往下一項的橋

```markdown
## 相關詞條
- [[男巫]]。儀式後的職業
- [[鬱結]]。儀式所釋放的物件
- [[附身]]。儀式結果產生的現象
- [[巫堂家的房屋]]。儀式常發生的地點
```

它把Librarying的`related`展開為五至十個連結，並附簡短關係說明。1,500項可形成5,000至10,000條連結，LOREBOOK還可補充部內連線。關係說明讓讀者能夠選擇下一項。

## 編輯註記與頁尾——不向讀者顯示的資料

```markdown
<!--
編輯註記:
- 原始來源: Vault [[神降儀式]]
- frontmatter同步日期: 2026-04-30
- 插畫規格: 儀式場景、全綵、可用跨頁
- 未確定設定: 儀式持續時間
- 寫作問題: 檢查附身描寫是否過強
- 狀態: ●
-->
```

HTML註釋儲存來源、同步日期、插畫、未確定設定、審查問題和六階段進度，在預覽、網頁與出版中隱藏。最後一行記錄位置、版本和更新日期。

```markdown
*- 02_第2部_世界法則/03_巫俗體系/01_神降儀式 / [LOREBOOK名] v0.3 / 2026-05-01*
```

## 直接轉換Librarying frontmatter

從13欄CSV機械對映：代表關鍵詞→`title`，分類→`category`，世界觀軸→`worldbuilding_axis`，別名→`aliases`，英文名→`english`，羅馬字→`romanization`，版本→`version`，翻譯註釋→`translation_notes`。定義進入摘要與匯入首句，詳細說明進入匯入，相關關鍵詞進入`related`，來源進入`sources`，`IDX`不用。LOREBOOK再增加三個欄位。

機械對映可在約五分鐘生成1,500個frontmatter完整的檔案，把人工一週的工作縮短到幾分鐘。

## 相同容器使差異更清晰

摘要、匯入、正文、相關詞條始終處於同一位置，讀者無需重新學習結構，注意力會直接落到內容。神降儀式與人物Kim Mi-seon使用同樣五部分時，儀式與人物、機制與人生、抽象與具體的差異會作為內容差異抵達讀者。

讀者能順暢從一項移動到下一項，一半來自資料的趣味，一半來自格式的一致。標準模板不是限製作者自由，而是建立讀者信任。
