# 第9章　短篇引用與多語言出版——收入正卷與排除在外的內容

設想書店的書架上擺著一本《Forgotten Realms Campaign Setting》（2001），旁邊還有《Realms of Mystery》（1998）、《Realms of Infamy》（1994）等十多本費倫短篇集。兩類書都是同一 IP 的資料，卻是不同的書。一類是城市、人物與勢力的目錄，另一類講述一個人物或一場事件。系列化封面讓人一眼看出它們屬於同一 IP，但書的身份與讀者用途彼此分開。CD Projekt 的《The Witcher》也採用同樣結構：《Compendium》（LOREBOOK）與薩普科夫斯基的短篇集分卷出版。定義一本書的，不只是書中收錄了什麼，也包括它**沒有**收錄什麼。

有六類資料不進入 LOREBOOK 正卷：本 IP 的短篇全文；若 IP 有遊戲規則，則包括規則數值；Librarying frontmatter 中標記為**未定**的詞條；標記為**廢棄**的詞條；遊戲元語言；以及內部整理資料所用的法則程式碼。排除這六類資料的理由，共同塑造了 LOREBOOK 的性質。

這裡要引入 SSOT（Single Source of Truth，單一事實源）。它規定一項資料由哪裡作出決定。若在 SSOT 層級中 LOREBOOK 低於規則書，規則數值就由規則書確定，LOREBOOK 只接收該決定。未定詞條若進入正文，LOREBOOK 本身也會變成未定稿，因此須在 Librarying 階段確定後再移入正文。廢棄詞條不出現在正文，可與廢棄理由一起列入附錄1術語表，或只在編輯注（HTML 註釋）中留下痕跡。“玩家擲骰子”這樣的遊戲元語言和 PHYS-M01 這樣的法則程式碼，也應進入規則書、附錄或編輯注等其他區域。

六類資料中體量最大的是短篇正文。即使一個 IP 運營五年後積累了五十至一百篇短篇，全文也不進入 LOREBOOK。本章前半討論原因及短篇的處理方式，後半討論把正卷轉移到其他語言的多語言出版。兩個主題放在同一章，是因為它們都在決定**什麼進入正卷，什麼留在正卷之外**。

## 短篇另冊出版的理由

假設一個 IP 有一百篇短篇。每篇平均7,000個韓文字元，總計70萬字，約為1,000至1,200頁。若 LOREBOOK 正卷約450頁，合訂後就是1,450頁。B5精裝本的書脊會超過80毫米，難以單手託舉。

僅憑篇幅，分卷就很自然；更決定性的理由是讀者的**用途**不同。

LOREBOOK 讀者為了檢索或通覽 IP 資料而翻書。他們要回答“這個人物是誰”“這座城市在哪裡”“這場事件發生在哪裡”等問題，並希望在一兩分鐘內從一個頁面取得資料。字典與指南兩種用途共存於一卷。

短篇集讀者則按閱讀時間體驗 IP 的故事。他們在一次閱讀中完整讀完一篇，在另一個時段再遇見下一篇。短篇集幾乎沒有字典用途，通讀方式也與 LOREBOOK 不同：通覽 LOREBOOK 是在詞條之間移動，通讀短篇集則是從頭到尾讀完一個故事。

若兩種用途裝訂在一起，讀者每次都要決定自己是在檢索、通覽 LOREBOOK，還是閱讀短篇。這種決策負擔會令整本書更難進入。分卷把負擔拆開：正卷負責檢索與通覽，短篇別冊負責故事閱讀。讀者只需選擇符合當下用途的書。

短篇別冊的出版規格宜與正捲成對。開本可與正卷同為B5，或採用便攜的A5；一百篇×每篇平均6至10頁，總計600至1,000頁；裝幀採用精裝或軟精裝，並考慮體量與重量；封面延續正卷的系列設計。豪華版可用書盒同時收納兩卷。別冊使用獨立 ISBN，並在自身附錄中提供按短篇、人物、視點和地點編制的索引。

## 短篇引用的三種形式——按深度區分的三個層級

短篇全文不進入正卷，但短篇引用會出現。它讓讀者在閱讀 LOREBOOK 時短暫接觸本 IP 的敘事資料，並按深度分為三種形式。

最深的一種是 *Pull Quote*（引用框）。當正文詞條高度依賴短篇資料時，把短篇的一個段落以引用框置於詞條中間。

```markdown
樸數巫金美善的一天從凌晨三點開始。她在家宅神壇前修整香條，
撥松一夜間凝固的香爐灰。這是金美善巫職一天中的第一項儀式。

> “那天的黎明也與其他黎明一樣。香燃得更久了一些，一縷煙
> 觸到神壇的天花板後散開。金美善在神壇腳下攤開手掌。
> 那一天，第一次有神觸碰了那隻手掌。”
> ——短篇集023《神壇的黎明》，首段

以神壇之香開始一天的景象，在本 IP 的巫俗家宅中普遍相似。
凌晨三點是一家人與神相遇的時間，也由此塑造了巫職的結構。
```

短篇的一段文字進入正文散文流，引用後的正文再承接並深入解釋。標準做法是引用約50至200個韓文字元，註明出處（“短篇集XXX《標題》，段落位置”），並在引用後立即展開說明。每一部約出現5至10次。

較輕的形式是在**短引文後加上別冊指引**。用一兩行引文點明聯絡，再附上“全文見短篇別冊§023”等說明。每一部約出現10至20次。

最輕的形式是**不直接引用，只指出語境**。例如：“金美善的降神儀式是短篇集023《神壇的黎明》的舞臺。”正文不依賴短篇，只告訴想閱讀故事的讀者該去哪裡。每一部約出現20至30次。

三種形式合計每一部約35至60次，一本十二部結構的 LOREBOOK 約有400至700處短篇引用。這一數量足以讓 IP 的短篇資料自然流經正卷，又不會吞沒正卷。

## 短篇與 LOREBOOK 發生衝突時

短篇與 LOREBOOK 可能出現資料衝突，例如人物履歷在短篇中是A，在 LOREBOOK 中卻是B。誰優先由 SSOT 層級決定。

標準順序是：角色表最高，其次是經主審確認的短篇，再次是 LOREBOOK，最後是 Librarying Vault 的散文。根據 IP 的性質，規則書也可能排在更高位置。

這意味著 LOREBOOK 必須服從短篇中已經確定的事實。LOREBOOK 不能否定短篇；若正文資料與短篇衝突，應更新 LOREBOOK 以與短篇一致。

但短篇未涉及的領域適用反向原則。若短篇沒有描寫人物的童年，LOREBOOK 可以補足，併成為該資料的 SSOT。用一句話概括：**短篇寫到的部分以短篇優先，短篇未寫到的部分由 LOREBOOK 決定**。

正卷與別冊之間還要遵守雙向引用原則。正卷引用別冊，別冊也應引用正卷。短篇結束後的頁邊可寫：“本篇《神壇的黎明》也見於 LOREBOOK §2-3 巫俗體系；金美善的詳細資料見§9-1 巫俗人物群。”這樣，讀者便能從短篇前往 LOREBOOK，也能從 LOREBOOK 返回短篇。

《Forgotten Realms》短篇集與 LOREBOOK 三十年來的配合就是範例。自1988年第一本《Realms of Valor》以來，約二十本短篇集與 LOREBOOK 共同出版。短篇首頁會標上 *Setting: Waterdeep, 1372 DR* 這樣的座標，LOREBOOK 的深水城詞條則以引用框收錄相關片段。讀者可從故事轉向城市的詳細資料，也可從資料詞條轉向事件的完整故事。兩本書承載同一 IP 的兩種紋理，並彼此支撐。

## 多語言出版——建立在韓文正本之上

第二個主題，是在韓文 LOREBOOK 正本出版後將其轉移到其他語言。

原則有三項：**韓文正本優先**、**多語言版採用獨立週期**、**直接利用翻譯註**。

韓文正本完成至v1.0後才開始多語言出版。若兩項工作同時進行，韓文正本會受多語言版影響而偏移，多語言翻譯又追不上正本更新，資料便分裂為兩個分支。先完成正本，再為多語言版設定獨立週期，才能保證一致性。

多語言出版的核心資料是 Librarying frontmatter 的 `translation_notes` 欄位。Librarying 已預先填寫每個詞條的翻譯方針，如建議音譯、英文解釋與建議腳註，多語言製作可直接接收這些資料。

虛構 IP“○○”中“降神儀式”的翻譯註如下：

```yaml
translation_notes:
  - 建議_音譯: "Sinnaenim Uisik"
  - 英文_解釋: "spirit-descent ritual; the moment when a Mudang first hosts a deity"
  - 建議_首次出現加腳註: true
  - 無法簡單翻譯的原因: "Western 'shaman' is broader; Mudang is gender-specific"
```

這些資料在英文 LOREBOOK 中形成如下正文：

```markdown
The Sinnaenim Uisik¹ is the first ritual in which a Mudang receives
a deity. The ceremony lasts for several days, with family members
and neighbors present. ...

¹ Spirit-descent ritual. The Korean term Sinnaenim ("spirit-descent")
  Uisik ("ritual") refers specifically to the moment when a Mudang
  (Korean shaman) first hosts a deity. Western 'shaman' is broader;
  Mudang is gender-specific (often female).
```

音譯名稱出現在英文正文中，首次出現處新增腳註¹，預先寫好的解釋進入腳註。把韓文正本轉成多語言版並非從白紙開始，而是接收 Librarying 已設計好的資料。

目的在於節省時間並保持一致。若從頭處理，每個詞條需30分鐘至1小時，1,500個詞條合計750至1,500小時；接收預備資料後，可縮短到約5至10小時的審校。一致性也來自上游：Librarying 階段預先確定標準寫法，各語言版本即可共同複用，避免因作品或譯者而變化。

## 音譯、意譯與解釋——一個稱謂如何轉移

多語言出版中，每個詞條要在音譯、意譯和解釋三條路徑間作出選擇。市場上的三個例子很有啟發。

《哈利·波特》在同一作品內部就有明顯分流。韓文版對“Hogwarts”“Muggle”“Quidditch”採用音譯，對“Sorting Hat”按功能意譯，對“Diagon Alley”則使用解釋性的語音處理。接近專名或世界身份的資料傾向音譯，指向功能的資料傾向意譯。Librarying 翻譯註若預先確定這些分支，韓文、日文和中文版便能保持一致。

托爾金的例子更明確。他親自留下給譯者的《Guide to the Names in The Lord of the Rings》，規定某些稱謂如Frodo應音譯，另一些如Bag End應按意義翻譯。這份資料成為四十多種語言譯本的 SSOT。LOREBOOK 的 Librarying 翻譯註處在同一位置。

村上春樹作品也有類似的一致性。《挪威的森林》英譯本把“Watanabe”和“Gotanda”音譯，卻把“東京大學”解釋為 University of Tokyo。人物名採用音譯，機構和地名採用解釋，規則始終一致。

三個例子說明，翻譯實際上是**分類工作**。一個 IP 內的稱謂按種類分為音譯、意譯和解釋。分類良好，下一種語言可在已有版本上自然建立；分類失序，同一稱謂會在不同版本中變形，使讀者困惑。

## 多語言出版的三種形式

LOREBOOK 的多語言出版有三種形式。

最常見的是在韓文正卷之外另行出版英文版。兩卷採用相同設計與規格配對，通常在韓文正本完成後約6至12個月推出。

較輕的形式是正卷僅用韓文，但在卷末部分附錄（術語表、人名索引）並列英文寫法。讀者閱讀韓文時也能取得英文形式。工作約需1至2個月，因為英文寫法可直接來自 frontmatter 的 `english` 與 `romanization` 欄位。

規模最大的是正卷多語言同步出版，所需時間至少為韓文正本的兩倍。前兩種是標準方案；同步出版只建議用於體量巨大的 IP。

## 正文標記原則

韓文正本的規則很簡單：正文使用韓文；除必要的漢字或拉丁字母並記外，不直接插入外語正文；多語言寫法只在 frontmatter、術語表和後設資料層管理；其他語言版本使用獨立週期製作。

在“降神儀式”的例子中，正文只出現韓文稱謂。首次出現時可以在括號中簡短並列漢字與羅馬字，如“降神儀式（身降神 儀式, Sinnaenim Uisik）是……”。漢字只在相應 IP 確有需要時使用，羅馬字遵循韓國國語羅馬字表記法。Spirit Possession Ritual 這樣的英文意譯不進入韓文正文的首次出現處，只在製作英文版時進入英文正文。

收入什麼、排除什麼，決定了 LOREBOOK 的身份。當短篇全文、規則數值、未定詞條、遊戲元語言、法則程式碼和外語正文都有明確的外部去處，正卷所收資料的語調自然會顯現。一卷書明確自己的領域，把領域之外的資料交給短篇別冊、規則書或多語言別卷。書的分量不是由頁數決定，而是由領域的清晰度決定。
