# 第11章　Phase 4～6——書的材料轉化為書的時間

設想印刷廠的景象：紙張沉甸甸地落在手中，新印頁面散發油墨氣味，顯示器按小時呈現CMYK四色依次疊印。一批1,000冊LOREBOOK約用12至16小時落到紙上，再進入裝訂車間成為一本本書。最終印刷標誌第一輪結束；在到達終點前，還有四個月讓**書的材料**轉變為**書的產品**。

Phase 3結束時，LOREBOOK仍只是材料。1,500個Markdown已有正文和wikilink，但部與部之間尚未綜合驗證，卷末附錄仍空，也未轉換成印刷PDF或網站。

Phase 4用1個月驗證一致性，Phase 5用1個月製作卷末附錄，Phase 6用2個月製作出版版與公開網頁版。本章討論最後四個月。

## Phase 4——資料之間的一致性

Phase 4驗證1,500個已完成詞條之間的關係。Phase 3逐部檢查，Phase 4第一次綜合檢查**部與部之間**。5,000至10,000條wikilink使一個詞條的改動影響其他資料。要確認一部的更新是否符合他部引用、人物履歷是否與短篇一致、Librarying與LOREBOOK frontmatter是否相符。

第一步是**死鏈檢查**。抽取正文所有wikilink並與實際檔案對照。若有`[[樸數巫]]`卻無對應Markdown，就是死鏈。發現後可補充詞條、批次更新改名後的連結，或刪除錯誤引用。

第二步是**重複資訊檢測**。同一資料散見多處時，只保留一個主要位置，其餘改為wikilink。若“降神儀式”的定義同時出現於第2部§3和第9部§1的樸數巫詞條，就以前者為主，後者只引用`[[降神儀式]]`。

第三步是最精密的**SSOT層級違規檢查**。把LOREBOOK與上位SSOT——規則書、角色表、短篇——對照，衝突時更新LOREBOOK。自動檢查能發現姓名、年份、地點等明確衝突；性格、關係、動機等細微衝突需要人判斷。

第四步是**名詞、視點與勢力體系一致性**。確認第6章的雙座標對映，以及Librarying的標準詞條名在全文一致。

```text
grep -r "舊寫法" .      → 0條才PASS
grep -r "白龍國" .      → 廢棄寫法，0條才PASS
grep -r "神壇" .        → 檢查詞條名稱一致性
```

第五步是**Librarying frontmatter一致性**。第5章所述的八個繼承欄位必須完全相同。指令碼把Librarying 13列CSV與LOREBOOK frontmatter自動比較並列出偏差。

Phase 4有七項驗收線：死鏈0；重複資訊0（或明確標註有意重複）；SSOT違規0；名詞、視點、勢力grep違規0；frontmatter一致率100%；完成最終版角色表同步；完成100篇短篇一致性驗證。全部✓才進入Phase 5。產物是彙總結果的`_檢查清單.md`。

自動檢查約1至2小時，人的判斷與驗收約2至3周，總計約1個月。機器承擔約90%的機械工作，人處理微妙衝突。

## Phase 5——一個月的卷末附錄

Phase 5為1,500個正文詞條製作卷末資料。

建議附錄共八種，必備四種、推薦四種。必備為**術語表**（5至15頁，按順序排列200至500個詞條）、**索引**（3至10頁，按人物、地名、主題列頁碼）、**地圖集**（4至12頁，世界、地區、城市、維度地圖）、**年表總覽**（3至8頁，把正文事件按時間排列）。它們支撐讀者通讀後的檢索與參照。

推薦四種按IP選用：**NPC迷你卡**（25至50名主要人物的一句話描述）、**勢力一句話目錄**、**家譜與關係圖**、**短篇別冊指南**。人物比重高的IP重視迷你卡，有短篇資產的IP幾乎必須有別冊指南。

術語表與索引可自動生成。抽取1,500個frontmatter定義並排序，形成“關鍵詞＋定義＋參照部”；抽取正文的其他詞條引用並按人物、地點、主題分類，形成頁碼列表。純手工需2至3周，自動生成加人工審校約1周。

六步流程為：第12部§1地圖集，排列地圖與說明；§2年表總覽，從第3部自動抽取再人工排序；§3 NPC迷你卡，從第9部抽取並審校；§4術語表，由frontmatter生成並審校；§5索引，由正文生成，頁碼在排版後自動填入；以及符合IP的可選附錄。約1個月完成。

產物是第12部達到✓，必備四種和適用的推薦附錄全部完成。正文1,500詞條、最多八類附錄、100%確認的插圖齊備，正卷接近完成，Phase 6將其移入兩種媒介。

## Phase 6——轉換為兩種媒介

最後的Phase 6把正文與附錄分成出版版與公開網頁版，流程依次為網頁轉換、出版排版、印刷下單。

### 網頁轉換

第1章的**單一Markdown源**原則在此生效：同樣的1,500個檔案生成兩種媒介。MEJE IP首選Quartz，它把Obsidian Vault轉換為保留wikilink與圖譜檢視的靜態網站。

替代方案有每月8至16美元的Obsidian Publish，以及設計自由度更高但學習成本更大的Hugo、Eleventy、Astro、VitePress、Docusaurus。可部署到Cloudflare Pages、Vercel、Netlify或GitHub Pages。

先在`_網頁轉換指南.md`中確定生成器、目錄到URL的對映、SEO後設資料（`title`、`description`、`keywords`）、色板和字型。構建含1,500詞條的試點站，驗證正文、插圖、交叉引用、圖譜檢視；配置如`lorebook.example.com`的域名、託管與HTTPS；最後正式構建並部署。約需2至4周。

### 出版排版

LOREBOOK流程負責Markdown，InDesign或Affinity Publisher等外部工具把它轉換為印刷PDF。

選擇專業出版的InDesign、替代方案Affinity Publisher，或主要用於小說的Vellum，並應用B5精裝模板、五色色板、T01至T17頁面模板。frontmatter轉為頁面後設資料，散文轉為正文，wikilink轉為頁碼參照，同時放置插圖。

Phase 5生成的索引暫時沒有頁碼；排版確定分頁後，系統自動填入，省去人工錄入。

最後試印一冊，檢查色調、字型、邊距、插圖位置。發現問題就更新排版後再試印。約需1至2個月。即使由外部出版團隊排版，讀者也參與驗收和決定。

### 印刷下單

建議清單包括：比較三家印廠的小量、標準、大量報價；確認正文、環襯、封面的紙樣；決定黑箔、金箔、UV、壓凸等封面工藝；按排版PDF實測書脊；檢查CMYK轉換與輸出色彩配置；以出版社名義申請ISBN；印製封底條碼；確認交期、運輸、驗貨與再版條件。

CMYK是Cyan、Magenta、Yellow、Key黑四色印刷標準，色域不同於螢幕RGB，轉換與打樣必不可少。RGB插圖轉為CMYK後常會變暗，五色色板的強調色也須預先檢查。

Phase 6有兩個產物：部署到域名的公開靜態網站，以及約450頁的B5精裝書。兩者共同結束LOREBOOK第一輪。

## Phase 4、5、6的門檻

從Phase 3進入4，要求所有部全部詞條達到◕以上並已開始委託插圖，自動檢查與主審驗收共同執行。從4進入5，要求`_檢查清單.md`違規0且一致性驗證透過。從5進入6，要求必備四種和選定的推薦四種附錄全為✓、插圖100%確認。從Phase 6到出版，主審須確認試印本、ISBN、條碼與法律規格。

未透過就返回當前Phase，不建議臨時繞行。

## 虛構IP“○○”的Phase 4～6案例

Phase 4約1個月。發現12條死鏈，其中新增5個詞條、更新7條連結後歸零。重複檢測發現兩處“降神儀式”定義，改為以第2部§3為主，第9部§1只保留連結。發現與短篇023《神壇的黎明》1處SSOT衝突後更新LOREBOOK。名詞檢查確認廢棄名“白龍國”0次、現行名“冬柏國”一致，frontmatter一致率100%。同步了金美善樸數巫角色表的最終更新，並確認100篇短篇中30篇引用資料全部一致。產物是全✓的`_檢查清單.md`。

Phase 5約1個月。整理五張地圖及說明——梅里屯、舊巫堂街、海原餐飲街、靈界、黃泉路；按時序排列80起事件；製作35張NPC迷你卡；自動生成並審校1,200個一句話術語；按人物、地點、主題生成索引。推薦附錄含迷你卡、五個家譜與別冊指南，全部達到✓。

Phase 6約2個月。Quartz網頁轉換經試點後部署到`lorebook.example.com/○○`，約3周。外部團隊使用InDesign，按B5精裝約450頁和T01至T17模板轉換Markdown、自動填索引頁碼，並在試印後微調，約5周完成正式印刷。比較三家報價，確定紙張、封面、裝訂，申請ISBN與條碼，最終印製1,000冊。產物是上線的網站和1,000冊出版書。

## 第一輪的終點

Phase 6產物完成，第一輪結束，讀者終於能把自己的IP LOREBOOK放上書架。

但LOREBOOK並非製作一次就結束。IP運營期間會有新短篇、角色更新、Librarying Vault增量、多語言出版，LOREBOOK也隨之更新。第13章討論它作為活資料的運營。

此刻只需確認9至14個月的第一輪已經結束：約450頁的B5精裝書進入書架，同樣的資料也作為靜態網站公開。這是把一個IP資料裝訂為一本書的首個產品。
