# 제11장. Phase 4~6 - 책의 재료가 책으로 변환되는 시간

한 인쇄소 작업장의 풍경을 떠올려 봅니다. 종이의 무게가 한 손에 묵직하게 잡히고, 갓 인쇄된 페이지의 잉크 냄새가 코에 닿습니다. CMYK 4색의 잉크가 차례로 종이에 쌓이는 과정이 시계 옆 모니터에 시간 단위로 표시됩니다. 1,000부의 LOREBOOK이 한 회 인쇄로 약 12~16시간 만에 종이로 옮겨지고, 그 종이가 제본실로 들어가 한 권의 책으로 묶입니다. 이 마지막 인쇄 작업이 LOREBOOK 1차 주기의 끝이고, 그 끝에 닿기까지 본문 자료가 **책의 재료**에서 **책의 산물**로 바뀌는 4개월이 놓여 있습니다.

Phase 3이 끝난 시점의 LOREBOOK은 아직 **책의 재료**입니다.

1,500개의 .md 파일에 본문이 채워져 있고 항목 사이에 wikilink가 걸려 있지만, 부 사이의 정합성이 아직 통합 검증되지 않았고, 권말 부록이 비어 있으며, 출판 PDF로도 웹 사이트로도 변환되지 않은 상태입니다. 독자의 책장에 들어갈 한 권의 책이 되기까지 세 단계가 더 남아 있습니다.

Phase 4는 자료들 사이의 정합성을 검증하는 한 달이고, Phase 5는 권말 부록을 짜는 한 달, Phase 6은 출판본과 웹 공개본을 만드는 두 달입니다. 이 회는 그 마지막 4개월, 책의 재료가 책으로 변환되는 시간을 다룹니다.

## Phase 4 - 자료들 사이의 정합

Phase 4는 정밀한 단계입니다. 1,500 항목의 본문이 작성된 뒤 자료들 사이의 정합성을 검증하는 단계입니다.

Phase 3 동안 매 부 단위로 정합성 체크가 이루어졌지만, **부 사이의** 정합성은 Phase 4에서 처음 통합 검증을 받습니다. 자료의 양이 1,500 항목이고 각 항목 사이에 5,000~10,000개의 wikilink가 걸려 있어, 한 항목의 변경이 다른 항목의 정합성에 영향을 미칩니다. 한 부에서 갱신된 자료가 다른 부의 인용과 일치하는지, 한 인물의 이력이 단편과 LOREBOOK에서 동일한지, 라이브러링 frontmatter가 LOREBOOK frontmatter와 어긋나지 않는지를 점검합니다.

다섯 단계로 짜입니다.

첫째는 **죽은 링크 검사**입니다. LOREBOOK 본문에 등장하는 모든 wikilink가 실제 항목을 가리키는지 확인합니다. 본문에 `[[박수무당]]`이 등장하지만 박수무당 .md 파일이 없으면 죽은 링크입니다. 자동 검사는 단순합니다. 본문 grep으로 모든 wikilink를 추출한 뒤 그 wikilink가 가리키는 .md 파일이 존재하는지 대조합니다. 죽은 링크가 발견되면 누락 항목을 추가하거나, 표제어 명칭이 갱신된 경우 wikilink를 일괄 갱신하거나, 인용이 잘못된 경우 wikilink를 제거합니다.

둘째는 **중복 정보 탐지**입니다. 같은 자료가 여러 항목에 흩어져 있는지 검사합니다. 한 표제어의 정의가 두 항목에 동일하게 등장하면 하나를 주로 두고 다른 곳은 wikilink로 대체합니다. 가공 IP "○○"의 신내림 의식 정의가 2부 §3 무속 체계와 9부 §1 무속 인물군의 박수무당 항목에 모두 등장한다면, 2부 §3을 주로 두고 박수무당 항목에서는 `[[신내림 의식]]`으로 인용만 둡니다.

셋째는 **SSOT 위계 위반 검사**이고, 가장 정밀한 단계입니다. LOREBOOK 본문이 상위 SSOT(룰북·캐릭터 시트·단편)와 모순되지 않는지 확인합니다. LOREBOOK 본문 자료를 캐릭터 시트와 단편 자료에 대조하고, 충돌이 발견되면 LOREBOOK을 갱신해 상위 SSOT에 맞춥니다. 사람의 비중이 큰 검사입니다. 자동 검사가 명확한 충돌(인물 이름·연도·장소)은 잡지만, 미묘한 충돌(인물 성격·관계·동기)은 사람의 안목이 필요합니다.

넷째는 **명사·시점·세력 체계 정합성**입니다. 본 IP의 모든 자료에서 명사 표기가 일관된지 검사합니다. 6장의 두 좌표 매핑이 제대로 적용되어 있는지, 라이브러링 frontmatter의 표제어 명칭이 본문에서 일관되게 쓰였는지를 자동 grep으로 점검합니다.

```
grep -r "구 표기" .  → 0건이어야 PASS
grep -r "백룡국" .   → 폐기 표기, 0건이어야 PASS
grep -r "신단" .     → 표제어 명칭 일관성 확인
```

마지막은 **라이브러링 frontmatter 일치성** 검사입니다. LOREBOOK 항목의 frontmatter가 라이브러링 frontmatter와 100% 일치하는지 살핍니다. 5장에서 다룬 대로 라이브러링이 채운 8 필드는 그대로 받기 때문에 두 자료가 어긋나면 안 됩니다. 자동 대조 스크립트로 라이브러링 13컬럼 CSV와 LOREBOOK frontmatter를 비교하면 어긋나는 항목들이 자동으로 잡힙니다.

다섯 단계가 모여 Phase 4의 합격선을 만듭니다. 죽은 링크 0건, 중복 정보 0건(또는 의도적 중복 명시), SSOT 위계 위반 0건, 명사·시점·세력 체계 grep 0건, frontmatter 일치성 100%, 캐릭터 시트 v_최종 동기화 완료, 단편 100편 정합성 검증 완료. 이 일곱 항목이 ✓이면 Phase 5로 가는 게이트가 열립니다. 산출물은 일곱 항목의 검사 결과를 정리한 _체크리스트.md 파일입니다.

자동 검사는 약 1~2시간에 끝나고, 사람의 결정과 검수가 약 2~3주 걸려 합쳐서 약 1개월입니다. 자동이 90%를 받아 주고 사람이 미묘한 충돌과 결정을 잡아내는 협업입니다.

## Phase 5 - 권말 부록 한 달

Phase 5는 LOREBOOK의 부록을 작성하는 단계입니다. 본문 1,500 항목 외에 권말 자료들이 들어갑니다.

권장 부록은 필수 4종과 권장 4종을 더한 8종입니다. 필수 넷은 **용어집**(Glossary, 5~15p, 가나다 순 200~500 표제어), **색인**(Index, 3~10p, 인물·지명·주제별 페이지 색인), **지도집**(Maps Atlas, 4~12p, 전세계·지역·도시·차원계 지도 모음), **연대기 총람**(Timeline, 3~8p, 본문에 흩어진 사건의 시간순 일람)입니다. 이 넷이 LOREBOOK의 검색과 참조 기능을 짜 줍니다. 독자가 본권을 통독한 후 자료를 다시 찾을 때 이 부록이 있어야 합니다.

권장 넷은 독자의 IP에 따라 선택적으로 적용합니다. **NPC 미니카드**(주요 인물 25~50명의 한 줄 묘사), **세력 한 줄 카탈로그**(모든 세력의 한 줄 요약), **가계도·관계도**(핵심 가문과 세력의 관계 다이어그램), **별권 단편집 안내**(단편 별권 분리 시의 안내 페이지)입니다. 캐릭터 비중이 큰 IP(아트북형이나 만화 공식 설정집형)는 NPC 미니카드가 무거워지고, 단편 자산이 있는 IP는 별권 안내가 거의 의무에 가깝습니다.

필수 부록 넷 중 용어집과 색인은 자동 생성이 가능합니다. 용어집은 1,500 항목의 frontmatter 정의 필드를 모두 추출하고 가나다 순으로 정렬하면 한 표제어가 한 줄(대표 키워드 + 정의 + 참조 부)이 됩니다. 색인은 1,500 항목의 본문에서 다른 표제어 인용을 모두 추출하고 인물·지명·주제별로 분류하면 각 표제어가 등장 페이지 번호 목록을 가집니다. 자동 생성이 Phase 5의 작업 시간을 크게 줄여 줍니다. 사람이 손으로 짜면 약 2~3주가 걸리지만, 자동 생성에 사람 검수를 더하면 약 1주에 끝납니다.

작업 흐름은 여섯 단계입니다. 12부 §1 지도집(모든 지도 일러스트와 캡션을 권말에 정렬), §2 연대기 총람(3부 역사·연대기에서 자료를 자동 추출하고 사람이 정렬), §3 NPC 미니카드(9부 존재 도감의 인물 항목에서 자동 추출하고 사람이 검수), §4 용어집(frontmatter 자동 생성에 사람 검수), §5 색인(본문 자동 생성, 페이지 번호는 조판 후 자동 채움), 그리고 IP에 따라 선택하는 권장 부록입니다. 여섯 단계가 약 1개월에 끝납니다.

산출물은 12부 부록의 ✓ 상태 도달이고, 필수 부록 넷과 독자의 IP에 맞는 권장 부록이 모두 작성된 상태입니다. 이 시점에서 LOREBOOK 본권은 거의 완성됩니다. 본문 1,500 항목과 부록 8종이 모두 갖춰지고, 일러스트 발주가 100% 컨펌된 상태입니다. 다음 Phase 6에서 이 자료가 출판본과 웹 공개본 두 매체로 옮겨집니다.

## Phase 6 - 두 매체로의 변환

Phase 6은 LOREBOOK 작업의 마지막 단계입니다. 본문과 부록이 완성된 LOREBOOK이 두 매체(출판본과 웹 공개본)로 갈라져 들어갑니다. 흐름은 셋이고, 웹 변환과 출판 조판, 인쇄 발주가 차례로 진행됩니다.

### 웹 변환

1장에서 다룬 **단일 .md 소스** 원칙이 여기서 작동합니다. 같은 .md 파일 1,500개가 두 매체를 짜 줍니다. 메제 IP의 1순위 변환 도구는 Quartz입니다. 옵시디안 Vault를 정적 사이트로 변환해 주는 도구이고, Vault의 wikilink와 그래프 뷰가 그대로 웹에 옮겨집니다. 옵시디안을 핵심 도구로 사용하는 작업 흐름에서 Quartz가 자연스럽습니다.

다른 도구도 가능합니다. 옵시디안 Publish(공식 호스팅)는 월 8~16달러의 호스팅 비용이 들고, Hugo·Eleventy·Astro·VitePress·Docusaurus 같은 정적 사이트 생성기는 디자인 자유도가 높지만 학습 비용이 듭니다. 배포는 Cloudflare Pages, Vercel, Netlify, GitHub Pages 등에서 모두 가능합니다.

웹 변환의 흐름은 _웹변환지침.md 작성에서 시작합니다. 정적 사이트 생성기 결정, URL 매핑 규칙(디렉토리 → URL 경로), SEO 메타(title·description·keywords), 색깔 팔레트와 폰트 결정. 그 다음 시범 사이트 빌드로 1,500 항목 모두를 변환하고 본문·일러스트·교차 참조를 검증하고 그래프 뷰 동작을 확인합니다. 도메인과 배포 설정(예: lorebook.example.com), 호스팅 설정, HTTPS 인증서. 마지막으로 본 빌드와 배포. 이 흐름이 약 2~4주에 끝납니다.

### 출판 조판

LOREBOOK 작업이 .md 파일까지를 책임지고, 그 .md 파일을 출판 가능 PDF로 옮기는 작업은 InDesign이나 Affinity Publisher 같은 외부 도구가 담당합니다.

조판 도구를 정하고(InDesign이 전문 출판, Affinity Publisher가 대안, Vellum이 소설 위주) 템플릿을 적용합니다. B5 양장 표준 템플릿, 5색 팔레트, T01~T17 페이지 템플릿을 사용합니다. .md를 인쇄 PDF로 변환하면서 frontmatter 자료를 페이지 메타로, 본문 산문을 페이지 본문으로, wikilink를 페이지 번호 참조로 자동 변환합니다. 일러스트 배치와 5색 팔레트 적용도 이 단계에서 함께 이루어집니다.

색인의 페이지 번호가 자동으로 채워지는 부분이 흥미롭습니다. Phase 5에서 자동 생성된 색인은 페이지 번호가 비어 있는 상태로 짜이고, 조판 후 페이지 번호가 정해지면 그 번호가 색인에 자동으로 채워집니다. 사람이 손으로 색인 페이지 번호를 짜 넣는 작업이 사라집니다.

마지막은 시범 인쇄본 검수입니다. 1부를 시범 인쇄해 색조와 폰트, 여백, 일러스트 위치를 점검하고, 흠이 발견되면 조판을 갱신해 다시 시범 인쇄합니다. 이 흐름이 약 1~2개월에 끝납니다. 조판 작업 자체는 외부 출판팀의 영역이지만 검수와 결정에는 독자가 참여합니다.

### 인쇄 발주

이 글은 인쇄 발주 실무를 자세히 다루지 않지만 권장 체크리스트는 짚어 둡니다.

견적 3곳 비교(소량·표준·대량), 종이 재질 샘플 확인(본문·면지·표지), 표지 가공 결정(먹박·금박·UV·엠보싱), 책등 두께 실측(조판 PDF 기준), CMYK 전환과 출력 컬러 프로파일 검수, 출판사 명의 ISBN 발급, 뒤표지 바코드 인쇄, 납기 일정과 운송, 검수와 재쇄 조건이 그 목록입니다.

CMYK는 Cyan·Magenta·Yellow·Key(검정) 4색 잉크의 인쇄 표준입니다. 화면용 RGB와 색역이 달라서 인쇄 전 변환과 교정이 필수입니다. RGB로 짠 일러스트가 CMYK로 변환되면서 색이 눌리는 경우가 흔하고, 5색 팔레트의 강조색 같은 자료는 CMYK 변환 후의 색을 독자가 미리 점검해 둬야 합니다.

Phase 6의 산출물은 둘입니다. 하나는 웹 공개본으로, 정적 사이트가 도메인에 배포되어 독자가 인터넷 주소로 LOREBOOK을 읽을 수 있게 됩니다. 다른 하나는 출판본으로, B5 양장 약 450p의 책 한 권이 인쇄되어 독자의 책장에 들어갑니다. 둘이 함께 LOREBOOK 1차 주기의 끝을 이룹니다.

## Phase 4·5·6의 게이트

이전 회의 Phase 0~3 게이트에 이어 Phase 4·5·6의 게이트를 짚어 둡니다.

Phase 3에서 4로 진입하는 게이트는 모든 부의 모든 항목이 ◕ 이상이고 일러스트 발주가 시작된 시점에 열립니다. 자동 검사와 마스터 검수가 함께 작동합니다. Phase 4에서 5로 진입할 때는 _체크리스트.md의 위반이 0건이고 정합성 검증을 통과한 자동 검사 결과가 필요합니다. 매 Phase 사이의 진입 조건이 명시되어 있고, 게이트를 통과하지 못하면 해당 Phase로 되돌아갑니다. 임시 우회는 권장하지 않습니다.

## 1차 주기의 끝

Phase 6의 산출물이 도달하면 LOREBOOK 1차 주기가 끝납니다. 독자가 자기 IP의 LOREBOOK 한 권을 갖추고, 그 책을 책장에 세워 둘 수 있게 되는 단계입니다.

이 시점에서 한 가지를 짚어 둡니다. LOREBOOK은 한 번 만들고 끝나는 자료가 아닙니다. 본 IP가 운영되는 동안 새 단편이 나오고, 캐릭터가 갱신되고, 라이브러링 Vault가 증분 업데이트되고, 다국어 발간이 진행됩니다. 그때마다 LOREBOOK도 함께 갱신됩니다. 살아 움직이는 자료로서의 운영은 이 글의 13장에서 다룹니다.

지금은 9~14개월의 1차 주기가 끝났다는 점만 짚어 둡니다. B5 양장 약 450p의 책 한 권이 독자의 책장에 들어가고, 같은 자료가 도메인에 배포된 정적 사이트로도 공개됩니다. 한 IP의 자료가 한 권의 책으로 묶여 나오는 1차 산출물입니다.
