# MEJE Books 5개 언어 원고·번역·배포 계획

## 1. 목표

MEJE Books의 모든 UI와 공개 원고를 다음 다섯 언어로 제공한다.

- 한국어: `ko`
- 영어: `en`
- 일본어: `ja`
- 중국어 번체: `zh-Hant`
- 중국어 간체: `zh-Hans`

출판 순서는 **원전 확정 → 원고 퇴고 → MEJE Books 임포트 → 번역 → 언어별 검수 → 홈페이지 빌드·배포**로 고정한다.

## 2. 2026-07-12 현재 상태

### 서비스

- Astro 기반 `MEJEBooks` 프로젝트가 있다.
- `scripts/import-books.mjs`가 `_Code`의 원전을 `content/books` 아래로 임포트한다.
- 현재 공개 목록은 10권, 246장이다.
- 한국어 라우트와 영어 라우트(`/en`)가 있다.
- 영어는 도서 서지·UI만 일부 번역됐고, 본문 번역 파일은 확인되지 않았다.
- 영어 본문이 없으면 한국어 원문 안내를 보여 주는 폴백이 구현돼 있다.
- `ja`, `zh-Hant`, `zh-Hans`는 UI, 라우트, SEO `hreflang`, 본문 로더가 아직 없다.

## 3. 출판 단위와 상태 모델

각 도서와 장은 다음 상태를 가진다.

1. `draft-ko`: 한국어 퇴고 중
2. `locked-ko`: 번역 기준 원문 고정
3. `translated`: 기계·초번역 완료
4. `language-reviewed`: 언어별 문체·용어 검수 완료
5. `content-reviewed`: 의미·사실·표·링크 검수 완료
6. `publishable`: 한국어 해시와 번역어의 기준 해시가 일치하는 배포 가능본
7. `published`: 운영 홈페이지 게시 및 스모크 테스트 완료

한국어 `locked-ko` 후에 본문을 바꾸면 해당 장의 모든 번역은 `stale`로 돌린다.

## 4. 파일 구조

한국어 원전은 기존 디렉터리를 유지한다. 번역은 임포트된 장 슬러그를 영구 키로 삼아 다음처럼 두는 것을 기준으로 한다.

```text
content/books/<book-slug>/
  book.json
  chapters/<chapter-slug>.md
  translations/
    en/<chapter-slug>.md
    ja/<chapter-slug>.md
    zh-Hant/<chapter-slug>.md
    zh-Hans/<chapter-slug>.md
  translation-manifest.json
```

번역 파일 frontmatter:

```yaml
---
title: "..."
sourceLocale: ko
locale: en
sourceHash: "sha256:..."
translationStatus: language-reviewed
translator: "..."
reviewer: "..."
updatedAt: "YYYY-MM-DD"
---
```

`translation-manifest.json`은 장별로 원문 해시, 언어별 상태, 번역·검수자, 최종 검수일을 기록한다.

## 5. 원고 퇴고 게이트

번역은 도서별 한국어 원문이 다음 게이트를 통과한 뒤에만 시작한다.

- 장 수·순서·제목 확정
- 중복 부록·중복 사례 정리
- 숫자·인용·고유명사·연도 검증
- 용어와 영문 표기법 확정
- 도서 내·도서 간 핵심 개념 일치
- 표·코드·수식·이미지·링크 유효성
- 번역하지 말아야 할 제품명·조직명·상표 표시
- 저자 판단이 필요한 미확정 쟁점 0건 또는 번역자 주석으로 격리
- 원고 lint와 MEJE Books 임포트·빌드 통과

원고 퇴고 순서:

1. 분위기공학(3장): 작은 범위로 전 파이프라인 검증
2. FTUE(30장): 현재 퇴고가 가장 많이 진행된 저자 원고
3. 세계관으로 세계를 읽는 법(30장)
4. 뉴콘텐츠 BM과 IP 확장 경제(30장)
5. MEJE 프로세스 5권(79장)
6. Pride and Prejudice 세계관 데이터(74장)

## 6. 번역 전 공통 자산

다음 자산을 먼저 만들어야 246장의 용어가 흐트러지지 않는다.

- 5개 언어 통합 용어집
- 고유명사·상표·인명·작품명 표기표
- MEJE, LOREBOOK, Vault, Librarying, FTUE 등 핵심어 불번역·번역 규칙
- 저자별·도서별 문체 가이드
- 숫자·날짜·괄호·인용부호·단위 표기 규칙
- 요약·의역·주석 허용 범위
- 증거·출처·직접 인용 보존 규칙

중국어 번체와 간체는 문자 변환만으로 만들지 않는다. 한국어 원문에서 각각 번역하되, 통합 용어집에서 대만·홍콩권과 중국 대륙권 표기를 따로 관리한다.

## 7. 서비스 구현 단계

### 1단계: i18n 기반 일반화

- `Locale`을 5개 언어로 확장
- UI copy를 언어별 파일로 분리
- `/`, `/en`, `/ja`, `/zh-hant`, `/zh-hans` 라우트 제공
- 언어 선택기에 KO·EN·JA·繁體·简体 표시
- `hreflang`: `ko`, `en`, `ja`, `zh-Hant`, `zh-Hans`, `x-default`
- 언어별 통화·날짜·숫자 포맷은 `Intl` 사용

### 2단계: 번역 로더와 메타데이터

- `getChapter` 로케일을 5개로 확장
- 장 제목도 번역 frontmatter에서 로드
- 도서 제목·부제·설명을 코드 상수가 아닌 로케일 메타데이터로 이동
- 번역 미제공 장은 해당 언어로 '번역 준비 중'을 표시하고 한국어 원문 링크 제공
- 번역 있음·없음을 sitemap과 메타데이터에 정확히 반영

### 3단계: 번역 파이프라인

- 원문 해시 계산
- 변경된 장만 번역 `stale` 처리
- 마크다운 구조, 표, 링크, 코드, 이미지 보존 검사
- 숫자·고유명사·직접 인용 대조
- 용어집 위반 검사
- 번역 누락·한국어 잔류·잘못된 로케일 문자 검사

### 4단계: 홈페이지 검수·배포

- 5개 언어 홈·목록·도서·장 페이지 정적 빌드
- 모든 언어 전환 링크 유효성
- 장별 이전·다음 링크와 목차 활성 상태
- 대표 5장을 언어별로 렌더링해 레이아웃·표·긴 단어·CJK 줄바꿈 육안 검수
- 404, canonical, sitemap, `hreflang`, Open Graph 검증
- 성능·모바일·키보드 접근성 점검
- GitHub Pages 프리뷰 배포 후 스모크 테스트
- 통과 후 `main` 배포

## 8. 도서별 실행 방식

한 권을 다음 순서로 끝낸 뒤 다음 권으로 넘어간다.

1. 한국어 퇴고 및 `locked-ko`
2. 원전 → MEJE Books 임포트, 해시 등록
3. 통합 용어집에 해당 권 전용 용어 추가
4. `en` 번역·검수
5. `ja` 번역·검수
6. `zh-Hant` 번역·검수
7. `zh-Hans` 번역·검수
8. 5개 언어 빌드·렌더링·링크 검증
9. 해당 권 배포

첫 파일럿은 분위기공학 3장으로 한다. 파일럿이 통과하면 같은 규칙과 도구를 나머지 243장에 적용한다.

## 9. 완료 조건

다음이 모두 증명되어야 전체 작업이 완료된다.

- 10권 246장의 한국어 원전이 `locked-ko`
- 모든 장에 `en`, `ja`, `zh-Hant`, `zh-Hans` 번역본 존재
- 모든 번역본의 `sourceHash`가 현재 한국어 원문과 일치
- 5개 언어 UI에 한국어 하드코딩·누락 없음
- 모든 로케일 라우트가 정적 빌드됨
- 번어별 용어집·문체·숫자·고유명사 검수 통과
- 표·링크·이미지·코드 보존 통과
- `hreflang`·canonical·sitemap·40 실제 번역 존재 상태와 일치
- 현재 임포트와 새 임포트 간 장 누락·중복 없음
- 프로덕션 홈페이지에서 언어별 대표 페이지 스모크 테스트 통과
